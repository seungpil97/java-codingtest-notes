## 문제 1. 출석 번호 정리

### 문제 설명

N명의 학생이 출석 체크를 했다. 출석한 학생 번호가 순서대로 주어질 때, 출석하지 않은 학생 번호를 오름차순으로 출력하라. 전체 학생 번호는
1번부터 N번까지다.

**입력:**

```
5
1 3 3 5 1
```

**출력:**

```
2 4
```

**제약 조건:**

- 1 ≤ N ≤ 1,000
- 출석 번호는 1 이상 N 이하

---

### 풀이 전 제출

> ⚠️ 이번 풀이에서 생략됨. 다음부터 반드시 작성.

---

### 풀이 아이디어

1. HashMap으로 출석한 번호를 기록 (key: 번호, value: true)
2. 1~N을 순회하면서 map에 없는 번호 출력

---

### 테스트 케이스 설계

| 케이스        | 입력                | 예상 출력  | 이유                 |
|------------|-------------------|--------|--------------------|
| 최솟값        | `N=1 / 1`         | (빈 출력) | 1명이 1번 출석 → 결석자 없음 |
| 최댓값        | `N=5 / 1 2 3 4 5` | (빈 출력) | 전원 출석              |
| 경계 — 전원 결석 | `N=3 / 1 1 1`     | `2 3`  | 1번만 출석, 나머지 결석     |
| 경계 — 중복 입력 | `N=5 / 1 3 3 5 1` | `2 4`  | 중복 번호 정상 처리 확인     |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n)
- 공간복잡도: O(n)

---

### 버그 기록

| 버그    | 원인                         | 수정                                       |
|-------|----------------------------|------------------------------------------|
| 후행 공백 | `sb.append(i).append(" ")` | `if (!sb.isEmpty()) sb.append(" ")` 로 변경 |

---

### 최종 코드

```java
package phase0.week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.*;

public class Apr21AttendanceList {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    Map<Integer, Boolean> map = new HashMap<>();

    for (int i = 1; i <= n; i++) {
      int key = Integer.parseInt(st.nextToken());
      map.put(key, true);
    }

    StringBuilder sb = new StringBuilder();
    for (int i = 1; i <= n; i++) {
      if (!map.containsKey(i)) {
        if (!sb.isEmpty()) sb.append(" ");
        sb.append(i);
      }
    }

    System.out.println(sb);
  }
}
```

---

### 💡 자료구조 분석

**HashMap (존재 여부 확인)**

장점: key 존재 여부를 O(1)로 확인 가능. N이 커져도 성능 유지.

단점: 이 문제처럼 값이 항상 `true`인 경우엔 HashSet이 더 적합. `Map<Integer, Boolean>`은 불필요한 value를
저장함.

실무 사용처: 출석 관리 시스템, 중복 요청 필터링, Redis의 SET 명령어와 동일한 패턴.

> ✅ 개선 포인트: 존재 여부만 확인할 때는 `HashSet` 사용 (3주차 이후 적용)

---

### 패턴 노트

```
## Apr21AttendanceList — 출석 번호 정리

- 문제 유형: HashMap 존재 여부 확인
- 핵심 패턴: 입력을 Map에 기록 → 전체 범위 순회하며 없는 것 출력
- 내가 놓친 포인트: 후행 공백 처리 누락
- 다음에 나오면 접근 방식: 존재 여부만 필요 → HashSet / key-value 매핑 필요 → HashMap
```

## 문제 2. 메뉴 주문 집계

### 문제 설명

카페에서 하루 동안 주문된 메뉴 이름이 순서대로 주어진다. 주문 횟수 상위 2개 메뉴를 출력하라. 동점이면 알파벳 순으로 먼저인 것을 우선한다.

**입력:**

```
8
latte americano latte mocha americano latte mocha mocha
```

**출력:**

```
latte 3
mocha 3
```

**제약 조건:**

- 1 ≤ N ≤ 1,000
- 메뉴 이름은 소문자 영문자로만 구성, 길이 1 이상 20 이하

---

### 풀이 전 제출

> ⚠️ 이번 풀이에서 생략됨. 다음부터 반드시 작성.

---

### 풀이 아이디어

1. HashMap으로 메뉴별 주문 횟수 집계 (`getOrDefault` 패턴)
2. entrySet을 ArrayList로 변환
3. Comparator로 정렬 — 횟수 내림차순, 동점이면 알파벳 오름차순
4. 상위 2개 출력

---

### 테스트 케이스 설계

| 케이스        | 입력                                        | 예상 출력                   | 이유                      |
|------------|-------------------------------------------|-------------------------|-------------------------|
| 최솟값        | `N=2 / latte americano`                   | `americano 1` `latte 1` | 동점 → 알파벳 순 americano 먼저 |
| 최댓값        | `N=5 / latte latte latte latte americano` | `latte 4` `americano 1` | 압도적 1위, 2위 1개           |
| 경계 — 전부 동점 | `N=4 / a b c d`                           | `a 1` `b 1`             | 모두 1회 → 알파벳 순 상위 2개     |
| 경계 — 메뉴 2개 | `N=3 / latte latte mocha`                 | `latte 2` `mocha 1`     | 정확히 2개만 존재할 때 정상 동작 확인  |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n log n) — entrySet 정렬
- 공간복잡도: O(k) — k는 고유 메뉴 수

---

### 버그 기록

| 버그        | 원인                                       | 수정                               |
|-----------|------------------------------------------|----------------------------------|
| 상위 2개 미출력 | 1위만 찾는 로직으로 구현                           | entrySet 정렬 후 상위 2개 추출로 변경       |
| 2위 로직 오류  | `else if (value >= firstMaxValue)` 조건 실패 | 정렬 방식으로 전환                       |
| 람다 변수명    | `(a, b)` 사용                              | `(firstEntry, secondEntry)` 로 수정 |

---

### 최종 코드

```java
package phase0.week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.*;

public class Apr21MenuOrder {

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    Map<String, Integer> menuOrderMap = new HashMap<>();

    for (int i = 0; i < n; i++) {
      String order = st.nextToken();
      menuOrderMap.put(order, menuOrderMap.getOrDefault(order, 0) + 1);
    }

    List<Map.Entry<String, Integer>> list = new ArrayList<>(menuOrderMap.entrySet());

    list.sort((firstEntry, secondEntry) -> {
      if (!firstEntry.getValue().equals(secondEntry.getValue())) {
        return secondEntry.getValue() - firstEntry.getValue();
      }
      return firstEntry.getKey().compareTo(secondEntry.getKey());
    });

    for (int i = 0; i < 2 && i < list.size(); i++) {
      System.out.println(list.get(i).getKey() + " " + list.get(i).getValue());
    }
  }
}
```

### 💡 자료구조 분석

**HashMap + ArrayList + Comparator 조합**

장점: 빈도 집계(HashMap) → 정렬(Comparator) → 추출(index) 단계가 명확히 분리돼 가독성이 높음. 상위 K개 추출
문제에 범용적으로 쓸 수 있음.

단점: 전체 정렬이 O(n log n)이라 상위 2개만 필요할 때는 과한 면이 있음. N이 커지면 두 번 순회 O(n)로 1, 2위를 직접 찾는
게 더 효율적.

실무 사용처: 카카오 검색어 순위, 네이버 실시간 트렌드처럼 상위 K개를 주기적으로 집계할 때 이 패턴을 사용. Redis의 Sorted
Set도 내부적으로 유사한 구조.

## 개념 정리 — 모범답안

### Q1. HashMap은 언제 써요?

key-value 쌍이 필요할 때 사용한다. 대표적인 케이스는 두 가지다.

첫째, **빈도 카운팅** — 특정 값이 몇 번 나왔는지 셀 때. key는 값, value는 횟수.

```
"latte" → 3, "mocha" → 3, "americano" → 2
```

둘째, **매핑** — 어떤 값에 대응하는 다른 값을 빠르게 찾을 때. key로 O(1) 조회가 가능하다.

존재 여부만 확인하면 되는 경우엔 HashSet이 더 적합하다. value가 항상 `true`라면 HashMap을 쓸 이유가 없다.

---

### Q2. `getOrDefault`가 뭐예요?

HashMap에서 key가 없을 때 기본값을 반환하는 메서드다.

```java
map.getOrDefault(key, 기본값)
```

빈도 카운팅에서 핵심 패턴이다.

```java
// key가 없으면 0 반환 → +1 → put
map.put(order, map.getOrDefault(order, 0) +1);
```

`map.get(key)`는 key가 없으면 `null`을 반환해서 `NullPointerException`이
발생한다. `getOrDefault`는 이걸 방지한다.

---

### Q3. Comparator에서 왜 int를 반환해요?

정렬 순서를 결정하기 위해서다. 규칙은 아래와 같다.

```
음수 반환 → 첫 번째 인자가 앞으로
양수 반환 → 두 번째 인자가 앞으로
0 반환    → 순서 유지
```

내림차순을 구현할 때 `b - a`를 쓰는 이유가 여기 있다.

- a=3, b=2이면 `b - a = 2 - 3 = -1` (음수) → a가 앞 → 큰 값이 앞 → 내림차순

boolean을 반환하면 "앞인지 아닌지"만 표현할 수 있어서 동점 처리가 불가능하다. int를 쓰면 음수/0/양수 세 가지 상태로 순서를
정밀하게 제어할 수 있다.
