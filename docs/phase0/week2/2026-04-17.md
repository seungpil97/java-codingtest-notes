## 🟡 검증 문제. 팰린드롬 부분 문자열 개수

**클래스명:** `Apr17Verify`

### 문제 설명

알파벳 소문자로 이루어진 문자열 S가 주어진다.
문자열에서 팰린드롬인 부분 문자열의 개수를 구하라.
(같은 문자열이라도 시작/끝 위치가 다르면 다른 것으로 센다)

**입력:**

```
abc
```

**출력:**

```
3
```

**제약 조건:**

- 1 ≤ S 길이 ≤ 1,000
- 소문자 알파벳만 포함

---

### 문제 요약

- **한 줄 요약:** 알파벳 소문자로 이루어진 문자열 S가 주어질 때, 팰린드롬인 부분 문자열의 개수 출력
- **입력 / 출력 구조:** 문자열 1개 입력 → 팰린드롬 부분 문자열 개수 출력
- **헷갈리는 조건:** 같은 문자열이라도 위치가 다르면 다른 것으로 센다

---

### 풀이 전 제출 

```
1. 입력 크기 (N): 1,000
2. 목표 시간복잡도: O(n²)
3. 사용할 알고리즘 후보: 이중 for문 (시작점 i, 끝점 j) + 팰린드롬 확인
4. 선택한 이유: 시작점과 끝점 두 변수가 필요하므로 이중 for문 필요
```

---

### 풀이 아이디어

1. 시작점 `i`, 끝점 `j`로 모든 부분 문자열을 순회
2. 각 부분 문자열을 뒤집어서 원본과 같으면 팰린드롬
3. 팰린드롬이면 count++
4. 최종 count 출력

---

### 테스트 케이스 설계

| 케이스   | 입력             | 출력  | 이유                     |
|-------|----------------|-----|------------------------|
| 최솟값   | `"a"`          | `1` | 길이 1, 단일 문자            |
| 최댓값   | 길이 1,000짜리 문자열 | -   | 최대 입력 크기               |
| 경계 조건 | `"abc"`        | `3` | 모든 문자가 달라 길이 1짜리만 팰린드롬 |

---

### 시간복잡도 / 공간복잡도

- **시간복잡도:** O(n²) — 이중 for문으로 모든 부분 문자열 순회
- **공간복잡도:** O(n) — 부분 문자열 저장용 변수

---

### 버그 기록

| 버그                             | 원인                        | 수정                             |
|--------------------------------|---------------------------|--------------------------------|
| `c.substring(i, j+1)`          | `char`에는 `substring` 없음   | `input.substring(i, j+1)`으로 변경 |
| `j = 0` 시작                     | `j < i`일 때 `substring` 오류 | `j = i`부터 시작                   |
| `isPalindrome` 확인 후 count++ 누락 | 조건문 미작성                   | `if (isPalindrome) count++` 추가 |

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

/**
 * 🟡 Apr17Verify
 * 알파벳 소문자로 이루어진 문자열 S가 주어진다.
 * 문자열에서 팰린드롬인 부분 문자열의 개수를 구하라.
 * (같은 문자열이라도 시작/끝 위치가 다르면 다른 것으로 센다)
 */
public class Apr17Verify {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String input = br.readLine();

    int count = 0;
    for (int i = 0; i < input.length(); i++) {
      for (int j = i; j < input.length(); j++) {
        String sub = input.substring(i, j + 1);
        String reversed = new StringBuilder(sub).reverse().toString();
        boolean isPalindrome = sub.equals(reversed);

        if (isPalindrome) count++;
      }
    }

    System.out.println(count);
  }
}
```

### 💡 알고리즘 분석

**완전탐색 (Brute Force)**

**장점:**

- 모든 경우를 확인하므로 정답을 반드시 찾는다. N이 작을 때(≤ 1,000) 시간 안에 충분히 동작한다.

**단점:**

- N이 커질수록 O(n²), O(n³) 등으로 빠르게 느려진다. N ≥ 100,000이면 시간 초과 발생.

**실무 사용처:**

- 코딩테스트 초기 접근 전략으로 자주 사용. N이 작거나 최적화 방법이 떠오르지 않을 때 브루트포스로 먼저 통과 후 개선.
