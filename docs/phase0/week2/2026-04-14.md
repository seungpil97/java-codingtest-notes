## 문제 1. 모음 제거 후 뒤집기 🟢

### 문제 설명

문자열 s에서 모음(a, e, i, o, u)을 모두 제거한 뒤, 남은 문자열을 뒤집어서 반환한다.

- 입력: `hello world`
- 출력: `dlrw llh`
- 제약: 1 ≤ s의 길이 ≤ 1,000 / 영소문자와 공백으로만 구성

---

### 문제 요약

문자열 s가 주어짐 → 모음 제거 → 뒤집어서 반환

---

### 풀이 전 제출

| 항목       | 내용                       |
|----------|--------------------------|
| 입력 크기 N  | 1,000                    |
| 목표 시간복잡도 | O(n)                     |
| 알고리즘 후보  | for문 순회 후 모음 제거, reverse |
| 선택 이유    | 모음 확인을 위해 한 번은 순회 필요     |

---

### 풀이 아이디어

1. 문자열을 `charAt(i)`로 순회
2. 모음인 경우 `continue`로 건너뜀
3. 나머지 문자를 `StringBuilder`에 `append`
4. `sb.reverse()`로 뒤집어 출력

---

### 테스트 케이스 설계

| 케이스            | 입력         | 출력              |
|----------------|------------|-----------------|
| 최솟값            | `a`        | `` (빈 문자열)      |
| 최댓값            | 1,000자 문자열 | 모음 제거 후 뒤집힌 문자열 |
| 경계 — 공백만 있는 경우 | `   `      | `   `           |
| 경계 — 자음만 있는 경우 | `bcd`      | `dcb`           |

---

### 시간복잡도 / 공간복잡도

- 시간: O(n) — 문자열 한 번 순회
- 공간: O(n) — StringBuilder 사용

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Apr14RemoveVowelAndReverse {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String input = br.readLine();

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < input.length(); i++) {
      char c = input.charAt(i);
      if (c == 'a' || c == 'e' || c == 'i' || c == 'o' || c == 'u') {
        continue;
      }
      sb.append(c);
    }

    System.out.println(sb.reverse());
  }
}
```

---

### 💡 자료구조 / 알고리즘 분석

**StringBuilder**

장점:

- 가변 문자열이라 `append` 시 새 객체를 생성하지 않음. 루프 안에서 문자열을 누적할 때 `String +=` 대비 메모리·속도 모두
  유리

단점:

- thread-safe 하지 않음. 멀티스레드 환경에선 `StringBuffer` 사용 필요

실무 사용처:

- 카카오, 네이버 등 대용량 로그 처리 시 문자열 누적에 사용. MySQL 쿼리 빌더 내부에서도 동적 SQL 생성 시 유사한 패턴 활용

---

### 패턴 노트

- 문제 유형: String 처리
- 핵심 패턴: `charAt` 순회 → 조건 필터 → `StringBuilder` 누적 → `reverse()`
- 내가 놓친 포인트: 없음
- 다음에 나오면: 모음/자음 분리 문제 → `StringBuilder` + `continue` 패턴 바로 적용

---

### 회고

- 풀이 방향 즉시 잡힘. 구현도 깔끔하게 완료
- `"aeiou".contains()` vs 직접 비교 — 코테에서는 실수 없는 명확한 코드가 우선임을 재확인

## 문제 2. 두 문자열의 공통 문자 개수 🟡

### 문제 설명

문자열 a와 b가 주어질 때, 두 문자열에 공통으로 등장하는 문자의 종류 수를 반환한다.
같은 문자가 여러 번 등장해도 1번만 센다.

- 입력: `abcde` / `cdefg`
- 출력: `3`
- 제약: 1 ≤ a, b의 길이 ≤ 1,000 / 영소문자로만 구성

---

### 문제 요약

문자열 a, b가 주어짐 → 공통으로 등장하는 문자 종류 수 출력

---

### 풀이 전 제출 

| 항목       | 내용                                                  |
|----------|-----------------------------------------------------|
| 입력 크기 N  | 1,000                                               |
| 목표 시간복잡도 | O(n²)                                               |
| 알고리즘 후보  | for문 순회 + `contains` + 중복 제거                        |
| 선택 이유    | a를 순회하면서 b에 있는지 확인 필요. `contains`가 내부적으로 순회라서 O(n²) |

> O(n) 해법은 3주차 HashMap 학습 후 연결 예정

---

### 버그 기록

- **1차 시도**: `count++` 만 있고 중복 제거 로직 없음 → `aabcd` / `abcde` 입력 시 `a`가 두 번 카운팅되는
  버그
- **수정**: 이미 센 문자를 `StringBuilder`에 기록, `continue`로 건너뜀

---

### 풀이 아이디어

1. 문자열 a를 `charAt(i)`로 순회
2. `sb`(이미 센 문자 기록용)에 현재 문자가 있으면 `continue`
3. b에 현재 문자가 있으면 `sb`에 추가 + `count++`
4. 최종 `count` 출력

---

### 테스트 케이스 설계

| 케이스        | 입력                      | 출력      |
|------------|-------------------------|---------|
| 최솟값        | `a` / `a`               | `1`     |
| 최솟값        | `a` / `b`               | `0`     |
| 최댓값        | 알파벳 1,000자 / 알파벳 1,000자 | 최대 `26` |
| 경계 — 공통 없음 | `abcde` / `fghij`       | `0`     |
| 경계 — 중복 문자 | `aaaaa` / `aaaaa`       | `1`     |

---

### 시간복잡도 / 공간복잡도

- 시간: O(n²) — a 순회 × `b.contains()` 내부 순회
- 공간: O(n) — StringBuilder 사용

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Apr14CommonCharCount {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String a = br.readLine();
    String b = br.readLine();
    StringBuilder sb = new StringBuilder();

    int count = 0;
    for (int i = 0; i < a.length(); i++) {
      char c = a.charAt(i);

      if (sb.toString().contains(String.valueOf(c))) {
        continue;
      }

      if (b.contains(String.valueOf(c))) {
        sb.append(c);
        count++;
      }
    }

    System.out.println(count);
  }
}
```

---

### 💡 자료구조 / 알고리즘 분석

**중복 제거 패턴 (visited 기록)**

장점:

- 추가 자료구조 없이 `StringBuilder` 하나로 "이미 처리한 문자" 추적 가능. 구현이 단순하고 실수 여지가 적음

단점:

- `sb.toString().contains()` 가 루프마다 String 객체를 생성함. N이 커지면 비효율
- 근본적으로 O(n²). 대용량 데이터엔 HashMap/HashSet이 필요

실무 사용처:

- 소규모 데이터 필터링에서 사용. 실제 서비스에서는 중복 제거 시 HashSet을 사용 (카카오 추천 시스템의 중복 아이템 제거 등)

> ⚡ 3주차 HashMap 학습 후 이 문제를 O(n)으로 다시 풀어볼 것

---

### 패턴 노트

- 문제 유형: 완전탐색 + 중복 제거
- 핵심 패턴: 순회하면서 "이미 처리한 것" 기록 → `continue`로 건너뜀
- 내가 놓친 포인트: 처음엔 중복 제거 없이 단순 카운팅 → 버그 발생
- 다음에 나오면: "종류 수" 라는 표현이 나오면 → 중복 제거 로직 필수로 체크

---

### 회고

- "종류 수" 조건을 처음엔 간과해 버그 발생. 문제 요약 단계에서 이 조건을 더 명확히 잡아야 함
- `contains()` 가 O(n) 임을 인지하고 목표 복잡도를 수정한 것은 올바른 판단
- 3주차에서 HashMap으로 O(n) 버전 재구현 예정
