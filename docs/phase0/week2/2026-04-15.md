## 🟢 문제 1. 단어 뒤집기

### 문제 설명

문장이 주어진다. 문장 안의 각 단어를 뒤집어서 출력하라. 단어 사이 공백은 그대로 유지한다.

**입력:** `Hello World Java`  
**출력:** `olleH dlroW avaJ`

**제약 조건:**
- 문장 길이 ≤ 1,000
- 단어는 공백으로 구분
- 특수문자 없음

---

### 문제 요약

- **한 줄 요약:** 문장이 주어지고 각 단어를 뒤집어서 출력, 공백 유지
- **입력 / 출력 구조:** 입력 문자열, 출력 문자열
- **헷갈리는 조건:** 공백 유지

---

### 풀이 전 제출 + Claude 판정

| 항목 | 내용 |
|---|---|
| 입력 크기 (N) | 1,000 |
| 목표 시간복잡도 | O(n) |
| 사용할 알고리즘 | for문, reverse |
| 선택한 이유 | 전체 문자열 순회 필요 |

**판정: 적절한 방향**

---

### 풀이 아이디어

- `split(" ")`으로 단어 분리
- 각 단어를 `StringBuilder`로 reverse
- 단어 사이 공백 유지하며 결과 조합

---

### 테스트 케이스 설계

| 케이스 | 입력 | 예상 출력 |
|---|---|---|
| 최솟값 | `a` | `a` |
| 최댓값 | 1,000자 문자열 | 각 단어 뒤집힌 문자열 |
| 경계 조건 | `" "` (공백만 있는 경우) | ` ` |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n)
- 공간복잡도: O(n)

---

### 버그 기록

- 처음에 전체 문자열을 `reverse()` → `"avaJ dlroW olleH"` 출력됨
- 단어 단위로 reverse해야 함을 인지 후 수정

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Apr15ReverseWords {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String input = br.readLine();
    String[] words = input.split(" ");
    StringBuilder sb = new StringBuilder();

    for (int i = 0; i < words.length; i++) {
      sb.append(new StringBuilder(words[i]).reverse());
      if (i < words.length - 1) sb.append(" ");
    }

    System.out.println(sb);
  }
}
```

---

### 💡 자료구조 / 알고리즘 분석

**StringBuilder**

- **장점:** 문자열 반복 수정 시 String + 연산 대비 O(1) append. String은 불변이라 + 할 때마다 새 객체 생성 → N이 크면 O(n²)
- **단점:** thread-safe하지 않음. 멀티스레드 환경에선 StringBuffer 사용
- **실무 사용처:** 카카오 구현 문제 문자열 조립 시 필수. 백준 N ≥ 100,000 출력 시 println 대신 sb에 모아서 한 번에 출력

---

### 패턴 노트

- **문제 유형:** String 처리
- **핵심 패턴:** split → 단위별 처리 → 조합
- **내가 놓친 포인트:** 전체 reverse와 단어 단위 reverse 구분
- **다음에 나오면 접근 방식:** 단위가 있으면 split 먼저, 단위별 처리 후 조합

---

### 회고

처음에 전체 문자열을 reverse해서 틀렸다. 문제를 꼼꼼히 읽고 "각 단어를"이라는 조건을 먼저 확인했어야 했다. split을 쓰면 의도가 코드에 명확히 드러난다는 것을 배웠다.

## 🟡 문제 2. 두 수의 합 (인덱스 찾기)

### 문제 설명

정수 배열과 목표값 `target`이 주어진다. 배열에서 두 수를 더해 `target`이 되는 모든 쌍의 인덱스를 출력하라. (같은 인덱스 두 번 사용 불가)

**입력:**
```
5 7
2 4 3 1 5
```

**출력:**
```
0 4
1 2
```

**제약 조건:**
- 1 ≤ N ≤ 1,000
- 정답 쌍이 없으면 `-1` 출력

---

### 문제 요약

- **한 줄 요약:** 정수 배열과 타겟이 주어지고, 합이 타겟과 일치하는 두 수의 인덱스 쌍 출력
- **입력 / 출력 구조:** 정수 배열 및 타겟 입력, 인덱스 위치 출력
- **헷갈리는 조건:** 정답 쌍이 없으면 -1 출력

---

### 풀이 전 제출 + Claude 판정

| 항목 | 내용 |
|---|---|
| 입력 크기 (N) | 1,000 |
| 목표 시간복잡도 | O(n²) |
| 사용할 알고리즘 | 이중 for문 (완전탐색) |
| 선택한 이유 | 앞 인덱스부터 하나씩 고정 후 나머지 전체 순회 |

**판정: 적절한 방향**

---

### 풀이 아이디어

- 외부 for문: 기준 인덱스 i 고정
- 내부 while문: i+1부터 끝까지 순회
- 합이 target이면 인덱스 쌍 출력
- 한 번도 출력 안 했으면 -1 출력

---

### 테스트 케이스 설계

| 케이스 | 입력 | 예상 출력 |
|---|---|---|
| 최솟값 | N=1, `[5]`, target=5 | `-1` |
| 최댓값 | `[1,2,3,4,5]` target=6 | `0 4` / `1 3` |
| 경계 조건 | `[1,2,3,4,5]` target=10 | `-1` |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n²)
- 공간복잡도: O(1)

---

### 버그 기록

- 처음에 쌍을 찾으면 `break` → 같은 i에서 다른 쌍 놓침
- `break` 제거 후 while 끝까지 순회하도록 수정
- `-1` 출력 조건 누락 → `isTwoSum` 플래그 추가

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Apr15TwoSum {

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

    String[] input = br.readLine().split(" ");
    int n = Integer.parseInt(input[0]);
    int target = Integer.parseInt(input[1]);

    int[] nums = new int[n];
    StringTokenizer st = new StringTokenizer(br.readLine());
    for (int i = 0; i < n; i++) {
      nums[i] = Integer.parseInt(st.nextToken());
    }

    boolean isTwoSum = false;
    for (int i = 0; i < n; i++) {
      int currentNum = nums[i];
      int nextIndex = i + 1;

      while (nextIndex < n) {
        int sum = currentNum + nums[nextIndex];

        if (target - sum == 0) {
          System.out.println(i + " " + nextIndex);
          isTwoSum = true;
        }

        nextIndex++;
      }
    }

    if (!isTwoSum) {
      System.out.println(-1);
    }
  }
}
```

---

### 💡 자료구조 / 알고리즘 분석

**이중 for문 (브루트포스)**

- **장점:** 구현이 단순하고 직관적. N이 작을 때 충분히 빠름
- **단점:** N ≥ 100,000이면 O(n²) = 10^10 → 시간 초과. 이 경우 HashMap으로 O(n) 풀이 가능 (값 → 인덱스 저장 후 target - nums[i] 존재 여부 탐색)
- **실무 사용처:** N이 작은 완전탐색 문제에서 기본 접근. HashMap 풀이는 Phase 1 5주차에서 다룰 예정

---

### 패턴 노트

- **문제 유형:** 완전탐색 (이중 for문)
- **핵심 패턴:** 기준 고정 → 나머지 순회 → 조건 확인
- **내가 놓친 포인트:** break로 인해 같은 기준에서 여러 쌍 못 찾음. -1 출력 조건 누락
- **다음에 나오면 접근 방식:** 모든 쌍 탐색 시 break 쓰지 않기. 결과 없음 처리 플래그 항상 챙기기
