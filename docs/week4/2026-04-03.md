## 문제 1. 이진수 변환

### 문제 설명

10진수 양의 정수 N을 입력받아 재귀 함수를 이용해 2진수 문자열로 변환하여 출력한다.
반복문(`while`, `for`) 사용 금지 — 재귀로만 구현한다.

### 풀이 아이디어

- N을 2로 나눈 몫을 재귀 호출하고, 나머지를 뒤에 붙인다.
- 재귀 호출을 **먼저** 하고 `append`를 **나중에** 하면 MSB부터 올바른 순서로 출력된다.
- base case: N < 2이면 N 자체를 append하고 반환.

```
binaryConvert(10)
  binaryConvert(5)
    binaryConvert(2)
      binaryConvert(1) → append(1)
      append(0)         ← 2 % 2
    append(1)           ← 5 % 2
  append(0)             ← 10 % 2

결과: 1010 ✅
```

### 사용한 개념

- 재귀 (Recursion)
- StringBuilder

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(log N) — 2진수 자릿수만큼 재귀 호출
- 공간복잡도: O(log N) — 호출 스택 깊이

### 최종 코드

```java
package week4;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class April03BinaryConvert {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());

    StringBuilder sb = new StringBuilder();
    binaryConvert(n, sb);

    System.out.println(sb);
  }

  private static void binaryConvert(int n, StringBuilder sb) {
    if (n < 2) {
      sb.append(n);
      return;
    }
    binaryConvert(n / 2, sb);
    sb.append(n % 2);
  }
}
```

### 💡 재귀 (Recursion)

**장점:**

- 문제를 "더 작은 동일한 문제"로 쪼갤 수 있을 때 반복문보다 코드가 훨씬 짧고 의도가 명확하다. `n → n/2 → n/4 ...` 처럼
  분할 구조가 눈에 보이는 경우가 대표적이다.
- 트리·그래프처럼 깊이 우선으로 탐색해야 할 때 호출 스택이 탐색 경로를 자동으로 추적해줘서 별도 스택 자료구조 없이도 구현이 가능하다.

**단점:**

- 호출마다 스택 프레임이 쌓이기 때문에 N이 크면 `StackOverflowError`가 발생한다. Java 기본 스택 깊이는 약 5,000~
  10,000 수준이므로, N이 직접 재귀 깊이가 되는 구조라면 위험하다.
- 동일한 계산을 반복 호출하는 경우(피보나치 등) 지수 시간이 되어버리는데, 이때는 메모이제이션(DP)으로 보완해야 한다.

**실무 사용처:**

- **파일 시스템 탐색**: 운영체제가 디렉토리 트리를 순회할 때 재귀 구조를 사용한다. 디렉토리 안에 디렉토리가 중첩된 구조 자체가
  재귀적이기 때문이다.
- **JSON / XML 파싱**: Jackson, Gson 같은 라이브러리가 중첩 객체를 파싱할 때 재귀 호출로 깊이를 탐색한다.
- **병합 정렬**: 배열을 절반으로 나누고, 각 절반을 다시 동일한 방식으로 정렬하는 것이 핵심으로 재귀 구조다.

## 문제 2. 나무 자르기 (백준 2805번)

### 문제 설명

나무 N그루의 높이가 주어진다. 절단기 높이를 H로 설정하면 H보다 높은 나무는 `(나무 높이 - H)`만큼 잘려서 가져갈 수 있다.
집에 가져가려는 나무의 길이 합이 최소 M미터가 되도록 하는 절단기 높이의 최댓값을 구하라.

### 풀이 아이디어

- N ≤ 1,000,000 → O(n log n) 이하 필요 → 이분 탐색
- 탐색 범위: left = 0, right = 나무 최대 높이
- mid 높이로 잘랐을 때 합계가 M 이상이면 → 더 높게 자를 수 있으니 `left = mid + 1`, answer 갱신
- 합계가 M 미만이면 → 더 낮게 잘아야 하니 `right = mid - 1`

### 사용한 개념

- 이분 탐색 (Binary Search) — 파라메트릭 서치

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n log(maxHeight)) — 이분 탐색 O(log H) × 합계 계산 O(n)
- 공간복잡도: O(n)

### 버그 기록

- `treeCut(long m)` 파라미터를 `int`로 선언해 컴파일 에러 발생 → `long`으로 통일
- 불필요한 `Arrays.sort(tree)` 포함 → `calculateWood()`는 전체 순회라 정렬 불필요, 제거

### 최종 코드

```java
package week4;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class April03TreeCut {

  static int[] tree;

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String[] input = br.readLine().split(" ");
    int n = Integer.parseInt(input[0]);
    long m = Long.parseLong(input[1]);
    StringTokenizer st = new StringTokenizer(br.readLine());
    tree = new int[n];
    for (int i = 0; i < n; i++) {
      tree[i] = Integer.parseInt(st.nextToken());
    }

    treeCut(m);
  }

  private static void treeCut(long m) {
    long left = 0;
    long right = 0;
    for (int treeHeight : tree) {
      right = Math.max(right, treeHeight);
    }

    long answer = 0;
    while (left <= right) {
      long mid = (left + right) / 2;
      long wood = calculateWood(mid);

      if (wood >= m) {
        answer = mid;
        left = mid + 1;
      } else {
        right = mid - 1;
      }
    }

    System.out.println(answer);
  }

  private static long calculateWood(long height) {
    long total = 0;
    for (int treeHeight : tree) {
      if (treeHeight > height) {
        total += treeHeight - height;
      }
    }
    return total;
  }
}
```

## 문제 3. 숫자 카드 2 (백준 10816번)

### 문제 설명

N개의 숫자 카드를 갖고 있다. 정수 M개가 주어졌을 때, 각 정수를 숫자 카드로 몇 개 가지고 있는지 출력하라.

### 풀이 아이디어

- N ≤ 500,000 → O(n log n) 이하 필요 → 이분 탐색
- cards 배열을 정렬한 뒤, 각 쿼리마다 lowerBound / upperBound를 구해 개수를 계산한다.
- `count = upperBound(target) - lowerBound(target)`

```
인덱스:  0  1  2  3  4  5
cards: [ 1  2  2  3  4  4 ]

target = 2 를 찾는다면:
  lowerBound(2) → 2가 처음 나오는 인덱스 = 1
  upperBound(2) → 2보다 큰 수가 처음 나오는 인덱스 = 3
  count = 3 - 1 = 2 ✅
```

|    | lowerBound            | upperBound             |
|----|-----------------------|------------------------|
| 조건 | `cards[mid] < target` | `cards[mid] <= target` |
| 반환 | target 이상이 처음 나오는 인덱스 | target 초과가 처음 나오는 인덱스  |

### 사용한 개념

- 이분 탐색 — lowerBound / upperBound 패턴

### 시간복잡도 / 공간복잡도

- 시간복잡도: O((N + M) log N) — 정렬 O(N log N) + 쿼리 M개 × O(log N)
- 공간복잡도: O(N + M)

### 버그 기록

1. `while (target != cards[j])` 안에서 `if (target == cards[j])` → 절대 true가 될 수 없어
   count 항상 0
2. `Arrays.sort(nums)` → 쿼리 순서가 바뀌어 출력 순서 파괴
3. `j`가 외부에서 한 번만 초기화 → 루프마다 잘못된 위치 참조
4. StringBuilder 출력 시 공백 누락 → 숫자가 붙어서 출력됨

### 최종 코드

```java
package week4;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class April03CardCount {

  static int[] cards;

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    cards = new int[n];
    for (int i = 0; i < n; i++) {
      cards[i] = Integer.parseInt(st.nextToken());
    }

    int m = Integer.parseInt(br.readLine());
    st = new StringTokenizer(br.readLine());

    Arrays.sort(cards);

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < m; i++) {
      int target = Integer.parseInt(st.nextToken());
      int count = upperBound(target) - lowerBound(target);
      if (i > 0) sb.append(" ");
      sb.append(count);
    }

    System.out.println(sb);
  }

  private static int lowerBound(int target) {
    int left = 0, right = cards.length;
    while (left < right) {
      int mid = (left + right) / 2;
      if (cards[mid] < target) left = mid + 1;
      else right = mid;
    }
    return left;
  }

  private static int upperBound(int target) {
    int left = 0, right = cards.length;
    while (left < right) {
      int mid = (left + right) / 2;
      if (cards[mid] <= target) left = mid + 1;
      else right = mid;
    }
    return left;
  }
}
```

### 💡 이분 탐색 — lowerBound / upperBound

**장점:**

- 정렬된 배열에서 특정 값의 범위를 O(log n)에 찾을 수 있다. HashMap으로 카운팅하면 O(1)이지만, 범위 쿼리("3 이상 7
  이하인 카드 수")는 이분 탐색만 처리 가능하다.

**단점:**

- 반드시 정렬이 선행되어야 한다. 정렬 비용 O(n log n)이 포함되므로, 단순 존재 여부 확인이라면 HashSet이 더 빠르다.

**실무 사용처:**

- **MySQL B-Tree 인덱스**: `WHERE age BETWEEN 20 AND 30` 쿼리를 처리할 때 내부적으로
  lowerBound/upperBound와 동일한 방식으로 범위의 시작·끝 위치를 탐색한다.
- **Java `Arrays.binarySearch()`**: 내부적으로 동일한 원리이며, lowerBound/upperBound를 직접
  구현하면 중복값 처리까지 정확하게 제어할 수 있다.
