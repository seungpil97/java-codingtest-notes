## 문제 1. 랜선 자르기 🟡

### 문제 설명
K개의 랜선을 잘라서 N개 이상의 랜선을 만들 때, 만들 수 있는 랜선의 최대 길이를 구하라.

### 풀이 아이디어
길이(mid)를 이분 탐색의 대상으로 삼는다. `countCables(mid)`로 만들 수 있는 랜선 수가 N 이상이면 정답 후보로 저장하고 더 큰 값을 탐색한다.

### 사용한 개념
- Parametric Search
- 정답 범위 이분 탐색 (`answer = mid; left = mid + 1` 패턴)

### 시간복잡도 / 공간복잡도
- 시간: O(K log maxLen) — maxLen 최대 2³¹
- 공간: O(K)

### 버그 기록
- `arr = new int[n]` → `new int[k]` 로 수정 (n은 만들 랜선 수, k가 입력 랜선 수)
- `long left = 0` → `long left = 1` (0으로 나누기 방지)
- 반환 타입 `Object` → `long`
- `Arrays.sort()` 불필요 (전체 순회라 정렬 무의미)

### 최종 코드
```java
public class April02CableCut {
    static int[] cables;

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] firstLine = br.readLine().split(" ");
        int k = Integer.parseInt(firstLine[0]);
        int n = Integer.parseInt(firstLine[1]);

        cables = new int[k];
        for (int i = 0; i < k; i++) {
            cables[i] = Integer.parseInt(br.readLine().trim());
        }

        System.out.println(findMaxLength(n));
    }

    private static long findMaxLength(int targetCount) {
        long left = 1;
        long right = 0;
        for (int cable : cables) {
            right = Math.max(right, cable);
        }

        long answer = 0;

        while (left <= right) {
            long mid = (left + right) / 2;
            long cableCount = countCables(mid);

            if (cableCount >= targetCount) {
                answer = mid;
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return answer;
    }

    private static long countCables(long cutLength) {
        long total = 0;
        for (int cable : cables) {
            total += cable / cutLength;
        }
        return total;
    }
}
```

### 💡 Parametric Search (매개변수 탐색)

**장점:**
- "최적값을 직접 탐색"하는 대신 "조건 만족 여부"로 문제를 변환한다. 정답 범위가 2³¹이어도 약 31번만에 탐색이 끝난다.
- `check(mid)` 함수를 독립적으로 설계할 수 있어 로직이 명확하다.

**단점:**
- 정답이 단조증가/단조감소 성질을 가져야만 적용 가능하다. 성질이 없으면 이분 탐색 자체가 의미 없다.
- `check(mid)`가 O(N)이면 전체 O(N log N)으로 허용되지만, check 내부가 느리면 효과가 줄어든다.

**실무 사용처:**
- MySQL: 인덱스 range scan에서 시작/끝 지점을 찾을 때 B-Tree 내부적으로 이분 탐색을 사용한다. `WHERE price BETWEEN 100 AND 200` 같은 쿼리가 O(log N)에 동작하는 이유다.
- CDN 네트워크: 청크 크기를 이분 탐색으로 결정해 전송 효율을 최대화한다. 랜선 자르기와 구조가 동일하다.

## 문제 2. 공유기 설치 🔴

### 문제 설명
N개의 집에 C개의 공유기를 설치할 때, 공유기 간 최소 거리가 최대가 되도록 설치하고 그 최솟값을 구하라.

### 풀이 아이디어
"최솟값의 최댓값"을 구하는 문제 → 간격(mid)을 이분 탐색의 대상으로 삼는다.  
`countInstallable(mid)`: 간격을 최소 mid 이상으로 유지하면서 첫 번째 집부터 탐욕적으로 설치했을 때 몇 개 설치 가능한지 계산한다.

### 사용한 개념
- Parametric Search
- 탐욕법 (첫 번째 집에 무조건 설치 후 간격 기준으로 선택)
- `Arrays.sort()` (이번엔 정렬 필요 — 집 순서대로 순회해야 하므로)

### 시간복잡도 / 공간복잡도
- 시간: O(N log N) — 정렬 O(N log N) + 이분 탐색 O(N log D), D = 최대 좌표 차이
- 공간: O(N)

### 랜선 자르기와의 차이

| | 랜선 자르기 | 공유기 설치 |
|---|---|---|
| 이분 탐색 대상 | 자르는 길이 | 공유기 간격 |
| check 함수 | 전체 합산 | 탐욕적 순회 |
| `Arrays.sort()` | 불필요 | 필요 |
| 방향 | 길이 커질수록 개수 감소 | 간격 커질수록 설치 수 감소 |
| answer 저장 방향 | 동일 (`left = mid + 1`) | 동일 (`left = mid + 1`) |

### Q&A

**Q. `answer`가 한번 3으로 저장되면 나중에 2로 바뀔 수 있나?**

절대 바뀌지 않는다. `answer = mid; left = mid + 1` 패턴이 이를 보장한다.

```
mid=3 → true  → answer=3, left가 4 이상으로 이동
mid=5 → true  → answer=5 (3보다 큰 값으로만 갱신)
mid=7 → false → right 감소, answer 건드리지 않음

true  → left 올라감  → 다음 mid는 항상 더 큼  → answer 증가만 가능
false → right 내려감 → answer 건드리지 않음
```

answer는 단조증가만 가능하며 절대 감소하지 않는다.

**Q. `right` 초기값을 왜 `houses[n-1] - houses[0]`으로 잡나?**

탐색 대상이 "공유기 사이의 간격"이고, 가능한 최대 간격은 첫 번째 집과 마지막 집의 거리이기 때문이다. 배열이 정렬되어 있으므로 `houses[n-1] - houses[0]`이 정확한 상한이 된다.

### 최종 코드
```java
public class April02RouterInstall {
    static int[] houses;

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] input = br.readLine().split(" ");
        int n = Integer.parseInt(input[0]);
        int c = Integer.parseInt(input[1]);

        houses = new int[n];
        for (int i = 0; i < n; i++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            houses[i] = Integer.parseInt(st.nextToken());
        }

        Arrays.sort(houses);
        System.out.println(findMaxMinGap(c));
    }

    private static long findMaxMinGap(int c) {
        long left = 1;
        long right = houses[houses.length - 1] - houses[0];
        long answer = 0;

        while (left <= right) {
            long mid = (left + right) / 2;
            long installCount = countInstallable(mid);

            if (installCount >= c) {
                answer = mid;
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return answer;
    }

    private static long countInstallable(long minGap) {
        int count = 1;
        int lastInstalled = houses[0];

        for (int i = 1; i < houses.length; i++) {
            if (houses[i] - lastInstalled >= minGap) {
                count++;
              lastInstalled = houses[i];
            }
        }
        return count;
    }
}
```

## 회고

- Parametric Search의 핵심은 "정답을 이분 탐색의 대상으로 바꾸는 것"임을 체득했다.
- `right` 초기값은 문제마다 달라진다. 배열의 최댓값인지, 좌표 차이인지 상황에 맞게 판단해야 한다.
- `Arrays.sort()` 필요 여부는 check 함수가 "전체 합산"인지 "순서 있는 순회"인지에 따라 결정된다.
- `answer` 단조증가 보장 구조를 이해했다: true 조건에서 left가 올라가므로 이전 mid로 돌아올 수 없다.
