## 문제 1 - 올바른 괄호 문자열 만들기

### 문제
`(` 와 `)` 로만 이루어진 문자열이 주어졌을 때, 올바른 괄호 문자열로 만들기 위해 최소 몇 번 괄호를 추가해야 하는지 출력하시오.

### 풀이 아이디어
추가해야 하는 괄호는 두 종류다.
1. `)` 를 만났는데 매칭할 `(` 가 없을 때 → `(` 를 추가해야 함
2. 끝까지 다 봤는데 `(` 가 남아있을 때 → `)` 를 추가해야 함

`open` 변수로 현재 열려있는 `(` 개수를 추적하고, `count`로 추가 횟수를 누적했다.  
루프가 끝난 후 남은 `open` 값을 `count`에 더해서 출력했다.

### 사용한 개념
- `BufferedReader`
- `charAt()`
- `open` / `count` 변수로 괄호 상태 추적

### 시간복잡도
- O(n)

### 공간복잡도
- O(1)

---

### 버그 기록

#### `count += open`을 루프 안에 넣은 오류
```java
// 버그 코드 — 매 루프마다 open을 더함
for (...) {
    ...
    count += open;  // ❌
}

// 수정 코드 — 루프 끝난 후 한 번만
for (...) { ... }
count += open;  // ✅
```

**교훈:** 루프 종료 후 처리해야 할 로직은 반드시 루프 밖에 위치해야 한다.

---

### 최종 코드
```java
int count = 0;
int open = 0;

for (int i = 0; i < tokens.length(); i++) {
    char token = tokens.charAt(i);

    if (token == '(') {
        open++;
    } else {
        if (open > 0) {
            open--;
        } else {
            count++;  // 매칭할 ( 없음 → ( 추가 필요
        }
    }
}

count += open;  // 남은 ( 개수만큼 ) 추가 필요
System.out.println(count);
```

---

### 동작 예시

```
입력: ()))(

( → open=1
) → open=1>0 → open=0
) → open=0 → count=1
) → open=0 → count=2
( → open=1

끝: count += open=1 → count=3
출력: 3 ✅
```

---

### 면접 질문 1. 이 코드가 전체적으로 어떻게 동작하는지 설명해보세요.

#### 답변
문자열을 앞에서부터 순회하면서 `(`면 `open`을 증가시키고, `)`면 매칭할 `(`가 있으면 `open`을 감소시켰습니다.  
매칭할 `(`가 없으면 `(`를 추가해야 하므로 `count`를 증가시켰습니다.  
루프가 끝난 후 남은 `open` 개수만큼 `)`를 추가해야 하므로 `count += open`을 더해서 출력했습니다.

---

### 면접 질문 2. 이 코드의 시간복잡도와 공간복잡도는 어떻게 되나요?

#### 답변
문자열을 한 번만 순회하므로 시간복잡도는 O(n)입니다.  
추가로 사용하는 변수는 `count`, `open`뿐이므로 공간복잡도는 O(1)입니다.

---

### 회고
처음에 Stack을 사용하려 했지만 이 문제는 `open` 변수 하나로 충분히 해결할 수 있었다.  
자료구조를 먼저 꺼내기 전에 변수 하나로 해결할 수 있는지 먼저 판단하는 습관이 필요하다.  
또한 `count += open`을 루프 안에 넣어서 틀렸는데, 루프 종료 후 처리할 로직은 반드시 루프 밖에 위치해야 한다는 점을 확실히 익혔다.

## 문제 2 - 오큰수 구하기

### 문제
N개의 정수가 주어졌을 때, 각 원소의 오큰수를 출력하시오.  
오큰수란 해당 원소보다 오른쪽에 있으면서 처음으로 더 큰 수를 말한다.  
오큰수가 없으면 `-1`을 출력한다.

### 풀이 아이디어
스택에 인덱스를 저장하고, 현재 숫자가 스택 top 인덱스의 숫자보다 크면 그게 오큰수다.  
결과 배열을 `-1`로 초기화하고, 오큰수를 발견할 때마다 해당 인덱스에 값을 채웠다.  
루프가 끝나고 스택에 남은 인덱스는 오큰수가 없으므로 `-1`이 유지된다.

### 사용한 개념
- `BufferedReader`
- `Arrays.stream().mapToInt().toArray()`
- `Deque<Integer>` / `ArrayDeque` (인덱스 저장)
- `Arrays.fill()`
- 단조 스택 패턴

### 시간복잡도
- O(n) — 각 원소는 push / pop 각 한 번씩만 처리

### 공간복잡도
- O(n)

---

### 핵심 아이디어 — 스택에 값이 아닌 인덱스를 저장

같은 값이 여러 개일 때 값만 저장하면 어떤 원소인지 구분할 수 없다.  
인덱스를 저장하면 `result[stack.pop()] = arr[i]`로 정확히 위치를 특정할 수 있다.

---

### 동작 예시 (`3 5 2 7`)

```
result: [-1, -1, -1, -1]

i=0, arr[0]=3 → 스택 비어있음 → push(0)         스택: [0]
i=1, arr[1]=5 → 5 > arr[0]=3 → result[0]=5, pop
               스택 비어있음 → push(1)            스택: [1]
i=2, arr[2]=2 → 2 < arr[1]=5 → push(2)           스택: [1, 2]
i=3, arr[3]=7 → 7 > arr[2]=2 → result[2]=7, pop
               7 > arr[1]=5 → result[1]=7, pop
               스택 비어있음 → push(3)             스택: [3]

끝: 스택에 3 남음 → result[3]=-1 (이미 -1)
출력: 5 7 7 -1 ✅
```

---

### 최종 코드
```java
int n = Integer.parseInt(br.readLine());
int[] arr = Arrays.stream(br.readLine().split(" "))
    .mapToInt(Integer::parseInt)
    .toArray();
int[] result = new int[n];
Arrays.fill(result, -1);

Deque<Integer> stack = new ArrayDeque<>();

for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && arr[stack.peek()] < arr[i]) {
        result[stack.pop()] = arr[i];
    }
    stack.push(i);
}

StringBuilder sb = new StringBuilder();
for (int i = 0; i < n; i++) {
    if (i > 0) sb.append(" ");
    sb.append(result[i]);
}
System.out.println(sb);
```

---

### 면접 질문 1. 이 코드가 전체적으로 어떻게 동작하는지 설명해보세요.

#### 답변
결과 배열을 `-1`로 초기화하고, 스택에 인덱스를 저장했습니다.  
각 원소를 순회할 때 현재 값이 스택 top 인덱스의 값보다 크면, 그 인덱스의 오큰수가 현재 값이므로 결과 배열에 저장하고 pop했습니다.  
루프가 끝나고 스택에 남은 인덱스는 오큰수가 없으므로 초기값 `-1`이 유지됩니다.

---

### 면접 질문 2. 왜 스택에 값이 아닌 인덱스를 저장했나요?

#### 답변
오큰수를 찾으면 해당 원소의 결과 배열 위치에 값을 저장해야 하기 때문입니다.  
값만 저장하면 결과 배열의 어느 위치에 저장해야 할지 알 수 없습니다.  
인덱스를 저장하면 `result[stack.pop()] = arr[i]`로 정확한 위치에 바로 저장할 수 있습니다.

---

### 면접 질문 3. 이중 for문으로도 풀 수 있는데 왜 Stack을 사용했나요?

#### 답변
이중 for문은 각 원소마다 오른쪽 전체를 다시 순회하므로 시간복잡도가 O(n²)입니다.  
스택을 사용하면 각 원소가 push와 pop을 각 한 번씩만 수행하므로 O(n)으로 줄일 수 있습니다.  
n이 커질수록 두 방식의 성능 차이가 크게 벌어집니다.

---

### 회고
처음에 스택에 값을 직접 저장하려다가 결과 배열 위치를 특정할 수 없다는 문제를 깨달았다.  
스택에 인덱스를 저장하는 패턴은 오큰수, 히스토그램 등 여러 문제에서 반복적으로 나오므로 확실히 익혀두어야 한다.  
단조 스택(Monotonic Stack)이라는 이름으로 불리는 이 패턴은 O(n²)을 O(n)으로 줄이는 핵심 기법이다.

## 문제 3 - 큐 두 개로 스택 구현하기

### 문제
큐 두 개만 사용해서 스택의 `push`와 `pop`을 구현하시오.

### 풀이 아이디어
`push`는 `queue1`에 바로 추가한다.  
`pop`은 `queue1`의 마지막 하나만 남기고 나머지를 `queue2`로 옮긴 뒤, 마지막 값을 꺼내서 반환한다.  
그 후 `queue2`의 내용을 다시 `queue1`으로 복원한다.

### 사용한 개념
- `BufferedReader`
- 여러 줄 입력 처리 (`while readLine`)
- `Queue<Integer>` / `LinkedList`
- `poll()` / `add()`

### 시간복잡도
- push: O(1)
- pop: O(n)

### 공간복잡도
- O(n)

---

### 버그 기록

#### `LinkedList`를 스택처럼 사용한 오류
```java
// 버그 코드 — Queue가 아닌 스택 동작 사용
private static final LinkedList<Integer> mainStack = new LinkedList<>();
mainStack.push(...);  // 스택 동작
mainStack.pop();      // 스택 동작

// 수정 코드 — Queue 두 개로 스택 구현
private static Queue<Integer> queue1 = new LinkedList<>();
private static Queue<Integer> queue2 = new LinkedList<>();
```

**교훈:** 문제 조건을 반드시 읽고 제약 조건을 지켜야 한다. 결과가 맞아도 조건을 어기면 오답이다.

---

### pop() 동작 원리

```
push 1, push 2, push 3 후
queue1: [1, 2, 3]

pop() 호출:
1. queue1 마지막 하나 빼고 나머지 queue2로 이동
   queue1: [3]   queue2: [1, 2]
2. queue1에서 3 꺼내서 반환
3. queue2 내용 다시 queue1으로
   queue1: [1, 2]  queue2: []

반환값: 3 ✅
```

---

### 최종 코드
```java
private static Queue<Integer> queue1 = new LinkedList<>();
private static Queue<Integer> queue2 = new LinkedList<>();

private static void push(int value) {
    queue1.add(value);
}

private static int pop() {
    while (queue1.size() > 1) {
        queue2.add(queue1.poll());
    }
    int result = queue1.poll();

    while (!queue2.isEmpty()) {
        queue1.add(queue2.poll());
    }
    return result;
}
```

---

### 면접 질문 1. 이 코드가 전체적으로 어떻게 동작하는지 설명해보세요.

#### 답변
push는 `queue1`에 바로 추가합니다.  
pop은 `queue1`의 마지막 하나만 남기고 나머지를 `queue2`로 옮긴 뒤, 마지막 값을 꺼내서 반환합니다.  
그 후 `queue2`의 내용을 다시 `queue1`으로 복원해서 다음 연산을 준비합니다.

---

### 면접 질문 2. pop의 시간복잡도가 O(n)인 이유는 무엇인가요?

#### 답변
pop할 때마다 `queue1`의 모든 원소를 `queue2`로 옮기고 다시 복원하는 과정이 필요하기 때문입니다.  
원소가 n개라면 이동 작업이 n-1번 수행되므로 O(n)입니다.

---

### 면접 질문 3. 왜 큐로 스택을 구현하면 pop이 비효율적인가요?

#### 답변
큐는 FIFO 구조라 맨 앞에서만 꺼낼 수 있고, 스택은 LIFO 구조라 맨 뒤에서 꺼내야 합니다.  
큐에서 마지막 원소에 접근하려면 앞의 원소들을 모두 다른 곳으로 옮겨야 해서 O(n)이 됩니다.  
반대로 스택 두 개로 큐를 구현하면 amortized O(1)로 더 효율적으로 만들 수 있습니다.

---

### 회고
조건을 제대로 읽지 않아서 `LinkedList`를 스택처럼 쓰는 실수를 했다.  
결과가 맞아도 제약 조건을 어기면 의미가 없다는 점을 다시 확인했다.  
큐 두 개로 스택을 구현하는 원리 — 마지막 하나 빼고 나머지를 옮겼다가 복원하는 패턴을 확실히 익혔다.
