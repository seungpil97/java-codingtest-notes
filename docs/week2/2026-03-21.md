## 문제 1 - 같은 숫자는 싫어

### 문제
수열이 주어졌을 때, 연속으로 같은 숫자가 나오면 하나만 남기고 제거한 결과를 출력하시오.

### 풀이 아이디어
스택에 첫 번째 값을 push하고, 이후 값이 스택 top과 다를 때만 push하고 출력했다.

### 사용한 개념
- `Deque<Integer>` / `ArrayDeque`
- `peek()` — top 확인
- `StringBuilder` — 공백 포함 출력

### 시간복잡도
- O(n)

### 공간복잡도
- O(n)

---

### 버그 기록

#### 버그 1 — 입력 파싱 오류
```java
// 버그 코드 — 첫 줄 n을 안 버리고 바로 split
String[] input = br.readLine().split(" ");  // "6" split → ["6"] ❌

// 수정 코드
br.readLine();  // 첫 줄 n 버리기
String[] input = br.readLine().split(" ");  // ✅
```

#### 버그 2 — push와 append 대상 오류
```java
// 버그 코드
stack.push(topValue);              // ❌ 스택 top을 다시 push
sb.append(" ").append(topValue);   // ❌ top을 출력

// 수정 코드
stack.push(currentTop);            // ✅ 현재 읽은 값을 push
sb.append(" ").append(currentTop); // ✅ 현재 읽은 값을 출력
```

---

### 최종 코드
```java
br.readLine();
String[] input = br.readLine().split(" ");

Deque<Integer> stack = new ArrayDeque<>();
StringBuilder sb = new StringBuilder();

stack.push(Integer.valueOf(input[0]));
sb.append(stack.peek());

for (int i = 1; i < input.length; i++) {
    int currentTop = Integer.parseInt(input[i]);

    if (!stack.isEmpty()) {
        int topValue = stack.peek();
        if (topValue != currentTop) {
            stack.push(currentTop);
            sb.append(" ").append(currentTop);
        }
    }
}
System.out.println(sb);
```

---

### 💡 Stack vs 변수 선택 기준

이 문제는 사실 변수 하나로도 풀 수 있다.

```java
int prev = Integer.parseInt(input[0]);
sb.append(prev);

for (int i = 1; i < input.length; i++) {
    int current = Integer.parseInt(input[i]);
    if (current != prev) {
        sb.append(" ").append(current);
        prev = current;
    }
}
```

| | Stack 풀이 | 변수 풀이 |
|---|---|---|
| 시간복잡도 | O(n) | O(n) |
| 공간복잡도 | O(n) | O(1) |

**교훈:** 이전 값 1개만 필요하면 변수, 여러 개 필요하면 Stack을 사용해야 한다.

---

### 면접 질문 1. 왜 Stack을 사용했나요? 더 간단한 방법은 없나요?

#### 답변
이 문제는 바로 직전 값 하나만 기억하면 되므로 변수 하나로도 해결할 수 있습니다.  
Stack을 사용하면 공간복잡도가 O(n)이 되지만, 변수를 사용하면 O(1)로 줄일 수 있습니다.  
자료구조를 선택할 때는 얼마나 많은 이전 값을 기억해야 하는지를 먼저 판단해야 합니다.

---

### 회고
Stack을 습관적으로 꺼내지 않고 변수 하나로 충분한지 먼저 생각하는 습관이 중요하다.  
자료구조 선택 기준 — "이전 값 1개만 필요하면 변수, 여러 개 필요하면 Stack"을 기억해두어야 한다.

## 문제 2 - 프린터 큐 대기 시간

### 문제
N개의 작업이 큐에 들어있고, M번째 작업(0-index)이 완료되는 시점의 총 경과 시간을 출력하시오.

### 풀이 아이디어
index 0부터 M까지의 처리 시간을 누적 합산했다.

### 사용한 개념
- `BufferedReader`
- 첫 줄 파싱 — `split(" ")` 으로 N, M 분리
- 누적 합

### 시간복잡도
- O(n)

### 공간복잡도
- O(n)

---

### 버그 기록

#### 버그 1 — 첫 줄 파싱 오류
```java
// 버그 코드 — "5 2" 를 통째로 parseInt
int m = Integer.parseInt(br.readLine());  // ❌ NumberFormatException

// 수정 코드
String[] firstLine = br.readLine().split(" ");
int n = Integer.parseInt(firstLine[0]);  // 5
int m = Integer.parseInt(firstLine[1]);  // 2
```

#### 버그 2 — sum 초기값 오류
```java
// 버그 코드 — index 0을 초기값으로 쓰고 1부터 더함
int sum = Integer.parseInt(input[0]);
for (int i = 1; i <= m; i++) { ... }

// 수정 코드 — 0부터 m까지 전부 더함
int sum = 0;
for (int i = 0; i <= m; i++) {
    sum += Integer.parseInt(input[i]);
}
```

---

### 최종 코드
```java
String[] firstLine = br.readLine().split(" ");
int n = Integer.parseInt(firstLine[0]);
int m = Integer.parseInt(firstLine[1]);

String[] input = br.readLine().split(" ");

int sum = 0;
for (int i = 0; i <= m; i++) {
    sum += Integer.parseInt(input[i]);
}

System.out.println(sum);
```

---

### 면접 질문 1. 이 코드의 시간복잡도는 어떻게 되나요?

#### 답변
index 0부터 M까지 한 번 순회하므로 O(M)이고, 전체적으로는 O(n)입니다.

---

### 회고
첫 줄에 두 숫자가 공백으로 구분되어 있을 때 `split(" ")`으로 분리하는 패턴을 다시 확인했다.  
누적 합은 index 범위를 정확히 설정하는 것이 중요하다.

## 문제 3 - 스택 수열

### 문제
1부터 N까지의 수를 순서대로 스택에 push하고, 임의의 시점에 pop해서 수열을 만들 수 있는지 판별하시오.  
만들 수 있으면 push는 `+`, pop은 `-`로 출력하고, 만들 수 없으면 `NO`를 출력하시오.

※ 백준 1874번 — 실버 3

### 수열이란?
숫자들이 순서대로 나열된 것.  
예: `4 3 6 8 7 2 9 1` → 이 순서대로 pop이 되어야 한다.

### 풀이 아이디어
`next` 변수로 다음에 push할 숫자를 추적했다.  
수열의 각 숫자(target)가 나올 때까지 push하고, top이 target과 같으면 pop했다.  
top이 target보다 작으면 이미 지나친 숫자이므로 불가능하다.

### 사용한 개념
- `Deque<Integer>` / `ArrayDeque`
- `next` 변수로 push 추적
- `possible` 플래그
- `StringBuilder`

### 시간복잡도
- O(n) — 각 숫자는 push / pop 각 한 번씩만

### 공간복잡도
- O(n)

---

### 실패 케이스 두 가지

```
1. 수열에 N보다 큰 숫자가 있을 때
   N=8, target=9 → push 불가 → NO

2. 순서상 이미 지나친 숫자를 꺼내야 할 때
   스택 top이 target보다 크면 → 이미 지나침 → NO
```

---

### 버그 기록

#### N 범위 초과 push
```java
// 버그 코드 — N보다 큰 숫자도 push
while (next <= target) { ... }

// 수정 코드 — N까지만 push
while (next <= target && next <= n) { ... }
```

이렇게 하면 target이 N보다 크면 while이 멈추고 `stack.peek() != target` 으로 `possible = false` 처리된다.

---

### 동작 예시 (N=8, 수열: `4 3 6 8 7 2 9 1 5`)

```
target=4 → push(1,2,3,4) ++++, pop(4) -
target=3 → pop(3) -
target=6 → push(5,6) ++, pop(6) -
target=8 → push(7,8) ++, pop(8) -
target=7 → pop(7) -
target=2 → top=5 ≠ 2 → possible=false → NO ✅
```

---

### 최종 코드
```java
Deque<Integer> stack = new ArrayDeque<>();
StringBuilder sb = new StringBuilder();
int next = 1;
boolean possible = true;

for (int i = 0; i < n; i++) {
    int target = Integer.parseInt(sequence[i]);

    while (next <= target && next <= n) {
        stack.push(next++);
        sb.append("+\n");
    }

    if (stack.peek() == target) {
        stack.pop();
        sb.append("-\n");
    } else {
        possible = false;
        break;
    }
}

System.out.println(possible ? sb.toString() : "NO");
```

---

### 면접 질문 1. 이 코드가 전체적으로 어떻게 동작하는지 설명해보세요.

#### 답변
수열의 각 숫자를 target으로 읽으면서, target이 나올 때까지 next를 증가시키며 push했습니다.  
push 후 스택 top이 target과 같으면 pop하고, 다르면 이미 지나친 숫자이므로 불가능으로 판단했습니다.  
가능하면 +/-를 출력하고, 불가능하면 NO를 출력했습니다.

---

### 면접 질문 2. 왜 `next <= n` 조건을 추가했나요?

#### 답변
1부터 N까지만 push할 수 있기 때문입니다.  
이 조건이 없으면 target이 N보다 클 때 N을 초과해서 push하게 됩니다.  
`next <= n` 조건을 추가하면 while이 멈추고, top이 target과 달라서 자동으로 NO 처리됩니다.

---

### 면접 질문 3. 이 코드의 시간복잡도는 어떻게 되나요?

#### 답변
각 숫자는 push와 pop을 각 한 번씩만 수행합니다.  
for문과 while문이 중첩되어 있지만, next는 전체를 통틀어 N번만 증가하므로 전체 시간복잡도는 O(n)입니다.

---

### 회고
수열의 의미 — 숫자들이 순서대로 나열된 것 — 을 처음에 헷갈렸지만 예시를 통해 이해했다.  
핵심은 크기가 아니라 순서라는 점이다. N 범위 안에 있어도 순서가 맞지 않으면 불가능하다.  
`next` 변수로 push할 숫자를 추적하는 패턴은 스택 수열 문제의 핵심 아이디어다.
