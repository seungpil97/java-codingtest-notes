## 문제 1 - 스택으로 숫자 뒤집기

### 문제
정수 하나가 주어졌을 때, 스택을 사용해서 숫자를 뒤집어 출력하시오.  
단, 앞에 오는 0은 제거한다.

### 풀이 아이디어
문자열을 앞에서부터 순회하면서 각 자릿수를 스택에 push했다.  
pop하면 LIFO 덕분에 자동으로 역순이 된다.  
앞에 오는 0은 `leadingZero` 플래그로 처리했다.

### 사용한 개념
- `Deque<Integer>` / `ArrayDeque`
- `push()` / `pop()`
- `charAt(i) - '0'` — 문자를 숫자로 변환
- `leadingZero` 플래그

### 시간복잡도
- O(n)

### 공간복잡도
- O(n)

---

### 버그 기록

#### 버그 1 — 공백으로 split
```java
// 버그 코드 — "12345" 전체가 하나로 들어감
String[] tokens = br.readLine().split(" ");

// 수정 코드 — 자릿수 하나씩 처리
String input = br.readLine();
for (int i = 0; i < input.length(); i++) {
    stack.push(input.charAt(i) - '0');
}
```

#### 버그 2 — 모든 0을 제거
```java
// 버그 코드 — 중간에 있는 0도 제거해버림
if (Integer.parseInt(token) > 0)

// 수정 코드 — 뒤집은 후 앞에 오는 0만 제거
if (leadingZero && digit == 0) continue;
```

**교훈:** "앞의 0 제거"는 뒤집은 후 앞에서부터 0을 건너뛰는 방식으로 처리해야 한다.

---

### `charAt(i) - '0'` 원리

```
'0' = ASCII 48
'5' = ASCII 53
'5' - '0' = 53 - 48 = 5  → 숫자 5로 변환
```

---

### 최종 코드
```java
String input = br.readLine();
Deque<Integer> stack = new ArrayDeque<>();

for (int i = 0; i < input.length(); i++) {
    stack.push(input.charAt(i) - '0');
}

boolean leadingZero = true;
StringBuilder sb = new StringBuilder();

while (!stack.isEmpty()) {
    int digit = stack.pop();
    if (leadingZero && digit == 0) continue;
    leadingZero = false;
    sb.append(digit);
}

System.out.println(sb);
```

#### 답변
문자열을 앞에서부터 순회하면서 각 자릿수를 `charAt(i) - '0'`으로 변환해 스택에 push했습니다.  
스택은 LIFO 구조라 pop하면 마지막에 넣은 자릿수부터 나오므로 자동으로 역순이 됩니다.  
앞에 오는 0은 `leadingZero` 플래그로 건너뛰었습니다.

### 회고
문자열을 통째로 처리하려다가 자릿수 단위로 처리해야 한다는 점을 놓쳤다.  
`charAt(i) - '0'`으로 문자를 숫자로 변환하는 패턴을 확실히 익혔다.

## 문제 2 - 괄호 제거 후 숫자 합산

### 문제
숫자와 `+`, `-`, `(`, `)` 로 이루어진 수식 문자열이 주어진다.  
괄호 앞에 `-` 가 있으면 괄호 안의 부호가 반전된다.  
수식을 계산한 결과를 출력하시오.

### 풀이 아이디어
스택에 현재 부호 방향(`+1` 또는 `-1`)을 저장했다.  
`(` 를 만나면 현재 방향을 스택에 push하고, `)` 를 만나면 pop했다.  
숫자를 만나면 `stack.peek() * sign * 숫자값`을 result에 누적했다.

### 사용한 개념
- `Deque<Integer>` / `ArrayDeque`
- `push()` / `peek()` / `pop()`
- `c - '0'` — 문자를 숫자로 변환
- 부호 방향 관리

### 시간복잡도
- O(n)

### 공간복잡도
- O(n)

---

### 버그 기록

#### 버그 1 — `token`과 `c` 혼용
```java
// 버그 코드 — int token으로 char 비교 시도
int token = Character.getNumericValue(c);
if (Character.isDigit(token))  // ❌ token은 이미 int
if (token == '+')               // ❌ int와 char 비교

// 수정 코드 — char c로 판별, c - '0'으로 숫자 변환
char c = tokens.charAt(i);
if (Character.isDigit(c))
result += stack.peek() * sign * (c - '0');
```

**교훈:** 문자 판별은 `char`로, 숫자 계산은 `c - '0'`으로 변환해서 사용해야 한다.

---

### 동작 예시 (`1+(2-3)+(-1+2)`)

```
stack=[1], sign=1, result=0

1   → result += 1*1*1 = 1
+   → sign=1
(   → stack.push(1*1=1)    stack=[1,1]
2   → result += 1*1*2 = 3
-   → sign=-1
3   → result += 1*(-1)*3 = 0
)   → stack.pop()          stack=[1]
+   → sign=1
(   → stack.push(1*1=1)    stack=[1,1]
-   → sign=-1
1   → result += 1*(-1)*1 = -1
+   → sign=1
2   → result += 1*1*2 = 1
)   → stack.pop()          stack=[1]

출력: 1 ✅
```

---

### 최종 코드
```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
int sign = 1;
int result = 0;

for (int i = 0; i < tokens.length(); i++) {
    char c = tokens.charAt(i);

    if (Character.isDigit(c)) {
        result += stack.peek() * sign * (c - '0');
    } else if (c == '+') {
        sign = 1;
    } else if (c == '-') {
        sign = -1;
    } else if (c == '(') {
        stack.push(stack.peek() * sign);
    } else if (c == ')') {
        stack.pop();
    }
}

System.out.println(result);
```

---

### 면접 질문 1. 왜 스택에 부호 방향을 저장했나요?

#### 답변
괄호 앞에 `-`가 있으면 괄호 안의 모든 부호가 반전되기 때문입니다.  
괄호가 중첩될 수 있으므로 현재 괄호의 부호 방향을 스택에 저장하면, 괄호가 열릴 때 push하고 닫힐 때 pop해서 올바른 부호 방향을 유지할 수 있습니다.

---

### 면접 질문 2. `stack.push(stack.peek() * sign)`의 의미는 무엇인가요?

#### 답변
현재 괄호 안의 부호 방향을 계산해서 저장하는 코드입니다.  
예를 들어 바깥 방향이 `+1`이고 현재 sign이 `-1`이면 `1 * -1 = -1`이 저장됩니다.  
바깥 방향이 `-1`이고 현재 sign이 `-1`이면 `-1 * -1 = 1`이 저장되어 이중 부정이 처리됩니다.

---

### 회고
처음에 복잡한 상태 변수들로 접근했지만, 스택에 부호 방향만 저장하면 간단하게 해결된다는 점을 배웠다.  
`c - '0'`으로 문자를 숫자로 변환하는 패턴을 다시 한번 확인했다.

## 문제 3 - 히스토그램에서 가장 큰 직사각형

### 문제
N개의 막대로 이루어진 히스토그램에서 만들 수 있는 가장 큰 직사각형의 넓이를 출력하시오.

### 히스토그램 이해

각 숫자는 막대의 높이를 의미한다.

```
index:  0  1  2  3  4  5
높이:    2  1  5  6  2  3

6층              ■
5층           ■  ■
4층           ■  ■
3층           ■  ■     ■
2층     ■     ■  ■  ■  ■
1층     ■  ■  ■  ■  ■  ■
```

직사각형 넓이 = 선택한 구간의 **가장 낮은 높이 * 너비**

```
전체 [0~5] → min(2,1,5,6,2,3)=1 * 6 = 6
구간 [2~3] → min(5,6)=5 * 2 = 10  ← 최대!
```

---

### 풀이 아이디어
스택에 인덱스를 저장하고, 현재 막대가 스택 top보다 낮아지는 순간 직사각형을 계산했다.  
루프 끝에 가상 막대 0을 추가해서 스택에 남은 인덱스를 전부 처리했다.

### 핵심 논리

> 막대가 낮아지는 순간 = 이전 높은 막대들의 직사각형이 끝나는 순간

### 사용한 개념
- `Deque<Integer>` / `ArrayDeque`
- 인덱스 저장 패턴
- 가상 막대 0 트릭
- `Math.max()`

### 시간복잡도
- O(n) — 각 인덱스는 push / pop 각 한 번씩만

### 공간복잡도
- O(n)

### 난이도
- 백준 기준 플래티넘 5 (BOJ 6549)

---

### 너비 계산 공식

```java
int width = stack.isEmpty() ? i : i - stack.peek() - 1;
```

```
스택이 비어있으면 → 0번부터 현재 i까지 전체 → 너비 = i
스택에 남아있으면 → stack.peek() 다음부터 i 직전까지 → 너비 = i - stack.peek() - 1
```

---

### 동작 예시 (`2 1 5 6 2 3`)

```
i=0, h=2 → push(0)              stack:[0]
i=1, h=1 → 1<h[0]=2
           pop(0) h=2, w=1 → 2
           push(1)               stack:[1]
i=2, h=5 → push(2)              stack:[1,2]
i=3, h=6 → push(3)              stack:[1,2,3]
i=4, h=2 → 2<h[3]=6
           pop(3) h=6, w=1 → 6
           2<h[2]=5
           pop(2) h=5, w=2 → 10 ← 최대!
           push(4)               stack:[1,4]
i=5, h=3 → push(5)              stack:[1,4,5]
i=6, h=0 (가상)
           pop(5) h=3, w=1 → 3
           pop(4) h=2, w=4 → 8
           pop(1) h=1, w=6 → 6

출력: 10
```

---

### 최종 코드
```java
Deque<Integer> stack = new ArrayDeque<>();
int maxArea = 0;

for (int i = 0; i <= n; i++) {
    int currentHeight = (i == n) ? 0 : heights[i];

    while (!stack.isEmpty() && heights[stack.peek()] > currentHeight) {
        int height = heights[stack.pop()];
        int width = stack.isEmpty() ? i : i - stack.peek() - 1;
        maxArea = Math.max(maxArea, height * width);
    }
    stack.push(i);
}

System.out.println(maxArea);
```

---

### 면접 질문 1. 왜 스택에 값이 아닌 인덱스를 저장했나요?

#### 답변
너비를 계산하려면 막대의 위치(인덱스)가 필요하기 때문입니다.  
값만 저장하면 해당 막대가 어느 위치에 있는지 알 수 없어서 너비 계산이 불가능합니다.  
인덱스를 저장하면 `i - stack.peek() - 1`로 정확한 너비를 계산할 수 있습니다.

---

### 면접 질문 2. 가상 막대 0을 추가한 이유는 무엇인가요?

#### 답변
루프가 끝난 후 스택에 남아있는 인덱스들은 오른쪽에 자신보다 낮은 막대가 없어서 처리되지 않은 것들입니다.  
높이 0인 가상 막대를 추가하면 모든 남은 막대보다 낮으므로 while 루프가 돌면서 스택을 전부 비워줍니다.
