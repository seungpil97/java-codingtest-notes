## 🟢 문제 1. 중복 없는 번호판

### 문제 설명

주차장 입차 기록이 담긴 정수 배열이 주어진다. 같은 차량이 여러 번 입차될 수 있다. 입차한 차량 번호를 중복 없이, 입차 순서대로 출력하라.

```
입력:
7
1003 2047 1003 5521 2047 2047 9001

출력:
1003 2047 5521 9001
```

제약 조건: 1 ≤ N ≤ 1,000

---

### 문제 요약

- 한 줄 요약: 정수 배열에서 중복 제거 후 입력 순서대로 출력
- 입력 / 출력 구조: 정수 배열 → 중복 제거된 정수 배열
- 헷갈리는 조건: 없음

---

### 풀이 전 제출 + Claude 판정

```
1. 입력 크기 (N): 1,000
2. 목표 시간복잡도: O(n)
3. 사용할 알고리즘 후보: HashMap
4. 선택한 이유: 중복 제거를 위해
```

**판정: 적절한 방향입니다.**

---

### 풀이 아이디어

- HashMap으로 등장 여부 체크 (key: 번호, value: true)
- ArrayList로 순서 유지하며 추가
- 아직 등장하지 않은 번호만 두 자료구조에 등록

---

### 테스트 케이스 설계

| 케이스 | 입력               | 예상 출력        |
|-----|------------------|--------------|
| 최솟값 | N=1, `1003`      | `1003`       |
| 최댓값 | N=1000, 전부 다른 번호 | 1000개 그대로 출력 |
| 경계  | 전부 같은 번호         | 번호 1개만 출력    |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n) — 배열 1회 순회
- 공간복잡도: O(n) — HashMap + ArrayList

---

### 버그 기록

- 초기 제출: LinkedHashSet 사용 → 이번 주 주제(HashMap+ArrayList)와 맞지 않아 재구현

---

### 최종 코드

```java
public class Apr20UniqueCarNumber {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    List<Integer> result = new ArrayList<>();
    Map<Integer, Boolean> seen = new HashMap<>();

    for (int i = 0; i < n; i++) {
      int num = Integer.parseInt(st.nextToken());
      if (!seen.containsKey(num)) {
        seen.put(num, true);
        result.add(num);
      }
    }

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < result.size(); i++) {
      if (i > 0) sb.append(" ");
      sb.append(result.get(i));
    }

    System.out.println(sb);
  }
}
```

---

### 💡 자료구조 분석

**HashMap**

장점: 키 기반 조회·삽입이 평균 O(1). 존재 여부 체크에 최적

단점: 순서 보장 안 됨 → 순서 필요 시 ArrayList 병행 사용

실무 사용처: 카카오 서버 로그 중복 요청 ID 필터링, 유저 방문 페이지 히스토리 중복 제거

**ArrayList**

장점: 삽입 순서 유지, 인덱스 접근 O(1)

단점: 중복 체크 O(n) → HashMap과 조합해서 보완

---

### 패턴 노트

- 유형: 중복 제거 + 순서 유지
- 핵심 패턴: HashMap(존재 체크) + ArrayList(순서 저장)
- 놓친 포인트: value가 의미 없을 때 true/1 아무거나 가능. 순수 존재 체크만이면 HashSet이 더 적합하나 이번 주 주제상
  HashMap 유지
- 다음에 나오면: "순서 유지 + 중복 제거" → HashMap + ArrayList 바로 떠올리기

---

### 회고

LinkedHashSet으로 먼저 풀었다가 이번 주 주제에 맞게 HashMap + ArrayList로 재구현. 자료구조 선택 시 문제 풀이뿐
아니라 학습 목적도 고려해야 함을 인식.

## 🟡 문제 2. 신고 횟수 집계

### 문제 설명

신고 기록이 담긴 문자열 배열이 주어질 때, 신고를 가장 많이 받은 학생의 이름과 횟수를 출력하라. 동점이면 알파벳 순으로 먼저인 학생을
출력한다.

```
입력:
8
kim lee park kim lee kim park park

출력:
kim 3
```

제약 조건: 1 ≤ N ≤ 1,000

---

### 문제 요약

- 한 줄 요약: 문자열 배열에서 가장 빈도 높은 값과 횟수 출력
- 입력 / 출력 구조: 이름 배열 → 최다 신고 학생 이름 + 횟수
- 헷갈리는 조건: 동점 시 알파벳 순

---

### 풀이 전 제출 + Claude 판정

```
1. 입력 크기 (N): 1,000
2. 목표 시간복잡도: O(n)
3. 사용할 알고리즘 후보: HashMap
4. 선택한 이유: 중복 키 있을 경우 value 증가하는 방식으로 빈도 집계
```

**판정: 적절한 방향입니다.**

---

### 풀이 아이디어

- HashMap으로 이름별 신고 횟수 집계 (`getOrDefault` 활용)
- 순회하며 최댓값 추출
- 동점 시 `compareTo`로 알파벳 앞선 이름 선택

---

### 테스트 케이스 설계

| 케이스     | 입력               | 예상 출력        |
|---------|------------------|--------------|
| 최솟값     | N=1, `kim`       | `kim 1`      |
| 최댓값     | N=1000, 모두 같은 이름 | `해당 이름 1000` |
| 경계 (동점) | kim 3, park 3    | `kim 3`      |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n) — 입력 순회 + 맵 순회
- 공간복잡도: O(k) — k는 고유 이름 수

---

### 버그 기록

- 초기 제출: 동점 처리 누락 → `value > maxValue` 조건만 있어 HashMap 순회 순서에 따라 결과 달라질 수 있음
- 수정: `value == maxValue && name.compareTo(maxKey) < 0` 조건 추가

---

### 최종 코드

```java
public class Apr20MostReported {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    Map<String, Integer> reportCountMap = new HashMap<>();

    for (int i = 0; i < n; i++) {
      String key = st.nextToken();
      reportCountMap.put(key, reportCountMap.getOrDefault(key, 0) + 1);
    }

    String maxKey = "";
    int maxValue = 0;

    for (String name : reportCountMap.keySet()) {
      int value = reportCountMap.get(name);
      if (value > maxValue || (value == maxValue && name.compareTo(maxKey) < 0)) {
        maxValue = value;
        maxKey = name;
      }
    }

    System.out.print(maxKey + " " + maxValue);
  }
}
```

---

### 💡 자료구조 분석

**HashMap (빈도 카운팅 패턴)**

장점: `getOrDefault`로 초기값 처리 간결. 키별 빈도 집계에 최적

단점: 순서 보장 안 됨. 최댓값 추출 시 전체 순회 필요 O(n)

실무 사용처: 네이버 검색어 빈도 집계, 카카오 메시지 수신 횟수 카운팅, Redis INCR 명령어와 동일한 패턴

---

### 패턴 노트

- 유형: 빈도 카운팅 + 최댓값 추출
- 핵심 패턴: `map.getOrDefault(key, 0) + 1` 로 빈도 누적
- 놓친 포인트: 동점 처리 누락. 최댓값 추출 시 항상 동점 조건 문제에서 확인할 것
- 다음에 나오면: 빈도 카운팅 → HashMap. 최댓값 추출 시 동점 조건(알파벳순, 인덱스순 등) 반드시 확인

---

### 회고

`getOrDefault` 패턴 자연스럽게 사용. 동점 처리를 놓쳤는데, 앞으로 최댓값·최솟값 추출 문제에서는 동점 조건을 제약 조건에서 먼저
확인하는 습관 필요.
