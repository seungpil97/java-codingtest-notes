## 문제 1. 병합 정렬 구현 `March27MergeSort`

### 문제 설명

정수 배열을 병합 정렬로 오름차순 정렬하여 출력. Arrays.sort() 사용 금지.

### 풀이 아이디어

1. 배열을 절반으로 재귀 분할 (left >= right 이면 탈출)
2. 왼쪽 절반, 오른쪽 절반 각각 재귀 정렬
3. 두 정렬된 구간을 두 포인터로 비교하며 temp 배열에 병합
4. temp를 원본 arr에 복사

### 사용한 개념

- 재귀 분할 정복
- 두 포인터 병합
- 임시 배열 복사

### 시간복잡도 / 공간복잡도

- 시간: O(n log n) — 분할 log n 단계 × 각 단계 n번 비교
- 공간: O(n) — merge마다 temp 배열 생성

### 버그 기록

- `mergeSort(arr, arr[left], arr[mid])` — 인덱스 대신 값을 넘긴 실수
- `arr = temp` — 로컬 변수만 바뀌고 원본 배열이 안 바뀌는 실수 → 직접 복사로 수정
- `if-continue` 3개로 중복 작성 → `if-else` 하나로 개선

### 최종 코드

```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class March27MergeSort {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());
    int[] nums = new int[n];
    for (int i = 0; i < n; i++) {
      nums[i] = Integer.parseInt(st.nextToken());
    }

    mergeSort(nums, 0, n - 1);

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < n; i++) {
      if (i > 0) sb.append(' ');
      sb.append(nums[i]);
    }
    System.out.println(sb);
  }

  static void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;

    int mid = (left + right) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
  }

  static void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;

    while (i <= mid && j <= right) {
      if (arr[i] <= arr[j]) {
        temp[k++] = arr[i++];
      } else {
        temp[k++] = arr[j++];
      }
    }

    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    for (int idx = 0; idx < temp.length; idx++) {
      arr[left + idx] = temp[idx];
    }
  }
}
```

### 면접 질문 & 답변

**Q. 병합 정렬의 시간복잡도가 O(n log n)인 이유는?**

A. 배열을 절반씩 나누므로 분할 단계는 log n번 발생합니다. 각 단계에서 전체 n개의 원소를 한 번씩 비교하며 병합하므로 단계당 O(n)
입니다. 따라서 전체 O(n log n)입니다.

**Q. 병합 정렬과 퀵 정렬의 차이는?**

A. 병합 정렬은 항상 O(n log n)을 보장하지만 O(n)의 추가 공간이 필요합니다. 퀵 정렬은 평균 O(n log n)이지만 최악의 경우
O(n²)이며 제자리 정렬이 가능합니다.

### 회고

- 재귀 구조 자체는 금방 이해했는데 merge() 내부 로직에서 막혔다
- `arr = temp`가 안 되는 이유(로컬 변수)를 이해하고 나니 배열 복사 개념이 명확해졌다
- `if-else` 하나로 줄이는 리팩토링을 통해 코드를 더 간결하게 작성하는 습관을 들였다

## 문제 2. 가장 큰 수 만들기 `March27LargestNumber`

### 문제 설명

N개의 비음수 정수를 이어 붙여 만들 수 있는 가장 큰 수를 출력.

### 풀이 아이디어

숫자를 문자열로 변환 후, 이어 붙인 결과를 비교하는 Comparator로 정렬.
`"34" + "3" = "343"` vs `"3" + "34" = "334"` → 343이 더 크므로 34가 앞.

### 사용한 개념

- Comparator 커스텀 정렬
- String.compareTo()
- 엣지 케이스 처리 (모두 0인 경우)

### 시간복잡도 / 공간복잡도

- 시간: O(n log n) — Arrays.sort
- 공간: O(n)

### 버그 기록

- `(a, b) -> a + b > b + a` — boolean 반환 불가, compareTo로 수정
- `System.out.println(sb)` 중복 출력 — 제거
- "000..." 케이스 미처리 → `result.charAt(0) == '0'` 체크 추가

### 최종 코드

```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class March27LargestNumber {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());
    String[] tokens = new String[n];
    for (int i = 0; i < n; i++) {
      tokens[i] = st.nextToken();
    }

    Arrays.sort(tokens, (a, b) -> (b + a).compareTo(a + b));

    StringBuilder sb = new StringBuilder();
    for (String token : tokens) {
      sb.append(token);
    }

    String result = sb.toString();
    System.out.println(result.charAt(0) == '0' ? "0" : result);
  }
}
```

### 면접 질문 & 답변

**Q. 왜 숫자를 그대로 비교하지 않고 문자열로 이어붙여 비교하나요?**

A. 단순히 숫자 크기로 정렬하면 잘못된 결과가 나옵니다. 예를 들어 3과 30을 비교하면 30이 크지만, 이어붙이면 330 < 303이므로
3이 앞에 와야 합니다. 이어붙인 결과 자체를 비교해야 올바른 순서를 구할 수 있습니다.

**Q. Comparator에서 음수/양수의 의미는?**

A. 음수를 반환하면 a가 앞, 양수를 반환하면 b가 앞에 배치됩니다. `a - b`는 오름차순, `b - a`는 내림차순이 되는 이유도 이
원리에서 나옵니다.

### 회고

- Comparator가 int를 반환해야 한다는 것을 처음 알았다
- `compareTo`의 반환값이 Comparator 규칙과 동일하다는 것을 이해했다
- 엣지 케이스(모두 0)를 스스로 떠올리지 못한 점이 아쉽다. 앞으로 0, 빈 배열, 최대값 케이스를 항상 체크하는 습관을 들여야겠다

## 문제 3. 국영수 `March27StudentSort` (백준 10825)

### 문제 설명

N명의 학생을 국어 내림차순 → 영어 오름차순 → 수학 내림차순 → 이름 오름차순으로 정렬.

### 풀이 아이디어

- `names[]`와 `scores[][]`를 따로 저장
- 인덱스 배열 `Integer[] idx`를 정렬하여 names, scores를 간접 참조
- Comparator 안에서 else if 체이닝으로 4가지 조건 처리

### 사용한 개념

- 인덱스 배열 간접 정렬 패턴
- 다중 조건 Comparator (else if 체이닝)
- String.compareTo() 사전순 비교

### 시간복잡도 / 공간복잡도

- 시간: O(n log n)
- 공간: O(n)

### 버그 기록

- 두 번째 if에서 바로 return하여 3, 4번째 조건에 도달 불가 → else if로 수정
- `names[a] - names[b]` String 빼기 시도 → compareTo로 수정

### 최종 코드

```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class March27StudentSort {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    String[] names = new String[n];
    int[][] scores = new int[n][3];

    for (int i = 0; i < n; i++) {
      StringTokenizer st = new StringTokenizer(br.readLine());
      names[i] = st.nextToken();
      scores[i][0] = Integer.parseInt(st.nextToken());
      scores[i][1] = Integer.parseInt(st.nextToken());
      scores[i][2] = Integer.parseInt(st.nextToken());
    }

    Integer[] idx = new Integer[n];
    for (int i = 0; i < n; i++) idx[i] = i;

    Arrays.sort(idx, (a, b) -> {
      if (scores[a][0] != scores[b][0]) return scores[b][0] - scores[a][0];
      else if (scores[a][1] != scores[b][1]) return scores[a][1] - scores[b][1];
      else if (scores[a][2] != scores[b][2]) return scores[b][2] - scores[a][2];
      else return names[a].compareTo(names[b]);
    });

    StringBuilder sb = new StringBuilder();
    for (int i : idx) {
      sb.append(names[i]).append('\n');
    }
    System.out.print(sb);
  }
}
```

### 면접 질문 & 답변

**Q. 인덱스 배열을 따로 만들어 정렬하는 이유는?**

A. names[]와 scores[][] 두 배열이 같은 학생 정보를 인덱스로 공유하고 있기 때문입니다. 두 배열을 동시에 정렬하는 것은
번거로우므로, 인덱스 배열만 정렬하고 출력 시 해당 인덱스로 두 배열을 참조하는 패턴을 씁니다.

**Q. 다중 조건 정렬에서 else if를 써야 하는 이유는?**

A. if만 쓰면 앞 조건이 false일 때 다음 if로 넘어가지 않고, 조건이 true인 시점에서 return되어 이후 조건에 도달하지 못할
수 있습니다. else if로 연결해야 앞 조건이 false일 때만 다음 조건을 검사합니다.

### 회고

- 인덱스 배열 패턴을 처음 써봤는데, 두 배열을 동기화할 필요 없이 깔끔하게 정렬할 수 있어서 유용했다
- else if 체이닝 실수를 통해 Comparator 내부 흐름을 더 명확히 이해했다
- Comparator의 오름차순/내림차순 공식(a-b, b-a)과 그 이유를 완전히 이해했다
