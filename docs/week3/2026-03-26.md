## 문제 1. 수 정렬하기 3 (백준 10989)

### 문제 설명
N개의 수를 오름차순으로 정렬하여 출력. N ≤ 10,000,000, 값 범위 1~10,000

### 풀이 아이디어
값의 범위가 1~10,000으로 제한되어 있으므로 카운팅 정렬 사용.
count[값]++ 로 빈도를 기록하고, 인덱스 순서대로 출력하면 자동으로 오름차순.

### 사용한 개념
- 카운팅 정렬: 값을 인덱스로 사용해 빈도 기록

### 시간복잡도 / 공간복잡도
- 시간: O(n + k) — n=입력 개수, k=값의 범위(10000)
- 공간: O(k)

### 최종 코드
```java
public class March26CountingSort {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());

    int[] count = new int[10001];
    for (int i = 0; i < n; i++) {
      count[Integer.parseInt(st.nextToken())]++;
    }

    StringBuilder sb = new StringBuilder();
    for (int value = 1; value <= 10000; value++) {
      for (int j = 0; j < count[value]; j++) {
        sb.append(value).append("\n");
      }
    }
    System.out.println(sb);
  }
}
```

### 면접 Q&A
- Q. 카운팅 정렬은 언제 쓰나요?
- A. 값의 범위(k)가 작고 자연수일 때 사용합니다. O(n+k)로 비교 기반 정렬 O(n log n)보다 빠르지만, 값의 범위가 크면 메모리 낭비가 심해집니다.

- Q. 중첩 루프인데 왜 O(n²)가 아닌가요?
- A. 안쪽 루프가 매번 n번 도는 게 아니라, 전체를 합쳐서 n번 돌기 때문입니다.

### 회고
- PriorityQueue로 먼저 풀었다가 카운팅 정렬로 개선
- 값을 인덱스로 쓰는 발상이 핵심

## 문제 2. 좌표 압축 (백준 18870)

### 문제 설명
N개의 좌표를 0부터 시작하는 상대적 순위로 변환. 중복은 같은 순위.

### 풀이 아이디어
정렬 + 중복 제거한 배열을 만들고, 각 원소의 위치를 이분탐색으로 찾음.

### 사용한 개념
- Arrays.copyOf, Arrays.sort
- Stream distinct() 로 중복 제거
- Arrays.binarySearch() 로 인덱스 탐색

### 시간복잡도 / 공간복잡도
- 시간: O(n log n)
- 공간: O(n)

### 최종 코드
```java
public class March26CoordinateCompression {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    int n = Integer.parseInt(br.readLine());
    StringTokenizer st = new StringTokenizer(br.readLine());
    int[] nums = new int[n];

    for (int i = 0; i < n; i++) {
      nums[i] = Integer.parseInt(st.nextToken());
    }

    int[] sorted = Arrays.copyOf(nums, n);
    Arrays.sort(sorted);
    sorted = Arrays.stream(sorted).distinct().toArray();

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < n; i++) {
      sb.append(Arrays.binarySearch(sorted, nums[i]));
      if (i < n - 1) sb.append(" ");
    }
    System.out.println(sb);
  }
}
```

### 면접 Q&A
- Q. 좌표 압축이란?
- A. 값의 범위가 클 때 상대적 순위로 변환하는 기법입니다. 정렬 후 중복 제거한 배열에서 이분탐색으로 인덱스를 찾으면 O(n log n)에 해결할 수 있습니다.

### 회고
- O(n²) 브루트포스로 먼저 접근 → 시간 초과 인식
- 정렬 + binarySearch 조합으로 개선

## 문제 3. K번째 수 (백준 11004)

### 문제 설명
N개의 수를 정렬했을 때 K번째 수 출력. N ≤ 5,000,000

### 풀이 아이디어
Arrays.sort() 후 nums[k-1] 접근.

### 사용한 개념
- Arrays.sort
- StringTokenizer (대용량 입력)

### 시간복잡도 / 공간복잡도
- 시간: O(n log n)
- 공간: O(n)

### 최종 코드
```java
public class March26KthNumber {
  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    StringTokenizer st1 = new StringTokenizer(br.readLine());
    int n = Integer.parseInt(st1.nextToken());
    int k = Integer.parseInt(st1.nextToken());
    StringTokenizer st = new StringTokenizer(br.readLine());
    int[] nums = new int[n];
    for (int i = 0; i < n; i++) {
      nums[i] = Integer.parseInt(st.nextToken());
    }

    Arrays.sort(nums);
    System.out.println(nums[k - 1]);
  }
}
```

### 면접 Q&A
- Q. K번째 수를 더 빠르게 구하는 방법이 있나요?
- A. QuickSelect 알고리즘을 사용하면 평균 O(n)에 구할 수 있습니다. 코딩테스트에서는 Arrays.sort()로 충분합니다.

### 회고
- 입력 파싱을 split() → StringTokenizer로 개선
