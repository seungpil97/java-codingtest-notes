## 문제 1 - 버블 정렬 직접 구현

### 문제
정수 배열이 주어졌을 때, 버블 정렬을 직접 구현해서 오름차순으로 정렬 후 출력하시오.  
(`Arrays.sort()` 사용 금지)

### 풀이 아이디어
인접한 두 원소 `arr[j]`와 `arr[j+1]`을 비교해서 앞이 크면 swap했다.  
매 라운드마다 가장 큰 수가 맨 뒤로 이동하므로 다음 라운드는 맨 뒤를 제외하고 비교했다.  
정렬과 출력을 분리해서 가독성을 높였다.

### 사용한 개념
- 이중 for문
- 인접 원소 비교 및 swap
- `String[]` → `int[]` 변환

### 시간복잡도
- O(n²)

### 공간복잡도
- O(1)

---

#### 버그 1 — 선택 정렬 방식으로 구현

```java
if (Integer.parseInt(tokens[i]) > Integer.parseInt(tokens[j]))

// 수정 코드 — 인접한 j와 j+1 비교 (버블 정렬)
if (arr[j] > arr[j + 1])
```

#### 버그 2 — 안쪽 루프 범위 초과
```java
// 버그 코드 — j+1이 범위 초과
for (int j = 0; j < arr.length - i; j++)

// 수정 코드
for (int j = 0; j < n - 1 - i; j++)
```

#### 버그 3 — 정렬 루프에서 출력까지 함께 처리
```java
// 버그 코드 — 마지막 원소 누락
for (int i = 0; i < n - 1; i++) {
    ...
    sb.append(arr[i]);  // n-1번만 출력
}

// 수정 코드 — 정렬과 출력 분리

```

**교훈:** 버블 정렬은 반드시 **인접한 j와 j+1**을 비교해야 한다. 정렬과 출력은 분리하는 게 깔끔하다.

---

### 버블 정렬 동작 원리

```
[3, 1, 4, 1, 5]

라운드 1: 3>1 swap → 1>4? No → 4>1 swap → 4>5? No → [1, 3, 1, 4, 5]  5 확정
라운드 2: 1>3? No → 3>1 swap → 3>4? No              → [1, 1, 3, 4, 5]  4 확정
라운드 3: 1>1? No → 1>3? No                          → [1, 1, 3, 4, 5]  3 확정
라운드 4: 1>1? No                                    → [1, 1, 3, 4, 5]  완료
```

---

### 최종 코드
```java
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - 1 - i; j++) {
        if (arr[j] > arr[j + 1]) {
            int swapValue = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = swapValue;
        }
    }
}
```

---

### 면접 질문 1. 버블 정렬이 어떻게 동작하는지 설명해보세요.

#### 답변
인접한 두 원소를 비교해서 앞이 크면 swap합니다.  
매 라운드마다 가장 큰 수가 맨 뒤로 이동하므로, 다음 라운드는 맨 뒤를 제외하고 비교합니다.  
n-1번의 라운드가 끝나면 정렬이 완료됩니다. 시간복잡도는 O(n²)입니다.

---

### 면접 질문 2. 왜 안쪽 루프가 `n-1-i`까지인가요?

#### 답변
매 라운드가 끝나면 가장 큰 수가 맨 뒤에 확정됩니다.  
i번째 라운드가 끝나면 뒤에서 i개가 이미 확정되므로, 그 부분은 비교할 필요가 없습니다.  
`n-1-i`까지만 비교하면 불필요한 비교를 줄일 수 있습니다.

---

### 회고
버블 정렬과 선택 정렬의 차이를 처음에 헷갈렸다.  
버블 정렬 = 인접한 두 원소 비교, 선택 정렬 = 최솟값 인덱스 찾아서 swap이라는 차이를 확실히 익혔다.  
정렬과 출력은 항상 분리하는 게 좋다.

## 문제 2 - 선택 정렬 직접 구현

### 문제
정수 배열이 주어졌을 때, 선택 정렬을 직접 구현해서 오름차순으로 정렬 후 출력하시오.  
(`Arrays.sort()` 사용 금지)

### 풀이 아이디어
매 라운드마다 `i+1`부터 끝까지 탐색해서 최솟값의 인덱스를 찾았다.  
루프가 끝난 후 `arr[i]`와 `arr[minIndex]`를 swap했다.

### 사용한 개념
- 이중 for문
- 최솟값 인덱스 저장 후 swap
- `String[]` → `int[]` 변환

### 시간복잡도
- O(n²)

### 공간복잡도
- O(1)

---

### 버그 기록

#### 버그 1 — 루프 중 swap
```java
// 버그 코드 — 탐색 중 swap하면 배열이 망가짐
if (minValue > arr[j]) {
    minValue = arr[j];
    arr[j] = arr[i];     // ❌ 탐색 중 배열 변경
    arr[i] = minValue;
}

// 수정 코드 — 인덱스만 저장하고 루프 후 swap
int minIndex = i;
if (arr[minIndex] > arr[j]) {
    minIndex = j;  // 인덱스만 갱신
}
// 루프 후
int swapValue = arr[i];
arr[i] = arr[minIndex];
arr[minIndex] = swapValue;
```

#### 버그 2 — 안쪽 루프 시작점
```java
// 버그 코드 — 0부터 시작하면 확정된 구간도 다시 비교
for (int j = 0; j < arr.length - 1; j++)

// 수정 코드 — i+1부터 시작
for (int j = i + 1; j < arr.length; j++)
```

#### 버그 3 — 마지막 원소 제외
```java
// 버그 코드
for (int j = i + 1; j < arr.length - 1; j++)  // 마지막 원소 제외

// 수정 코드
for (int j = i + 1; j < arr.length; j++)  // 마지막 원소 포함
```

**교훈:** 선택 정렬은 반드시 **인덱스**를 저장하고 루프가 끝난 후 swap해야 한다.

---

### 선택 정렬 동작 원리

```
[3, 1, 4, 1, 5]

라운드 1: i=0, minIndex=1(값1) → swap(0,1) → [1, 3, 4, 1, 5]
라운드 2: i=1, minIndex=3(값1) → swap(1,3) → [1, 1, 4, 3, 5]
라운드 3: i=2, minIndex=3(값3) → swap(2,3) → [1, 1, 3, 4, 5]
라운드 4: i=3, minIndex=3(값4) → swap 없음 → [1, 1, 3, 4, 5]
```

---

### 최종 코드
```java
for (int i = 0; i < arr.length; i++) {
    int minIndex = i;

    for (int j = i + 1; j < arr.length; j++) {
        if (arr[minIndex] > arr[j]) {
            minIndex = j;
        }
    }

    int swapValue = arr[i];
    arr[i] = arr[minIndex];
    arr[minIndex] = swapValue;
}
```

---

### 면접 질문 1. 버블 정렬과 선택 정렬의 차이는 무엇인가요?

#### 답변
버블 정렬은 인접한 두 원소를 비교해서 큰 수를 뒤로 밀어내는 방식입니다.  
선택 정렬은 최솟값의 인덱스를 찾아서 맨 앞과 swap하는 방식입니다.  
둘 다 O(n²)이지만 선택 정렬이 swap 횟수가 더 적어서 실제 연산은 더 적습니다.

---

### 회고
최솟값을 찾으면서 동시에 swap하는 실수를 했다.  
선택 정렬의 핵심은 인덱스를 저장하고 루프가 끝난 후 swap한다는 점이다.  
루프 중에 배열을 수정하면 탐색 결과가 망가질 수 있다는 점을 확실히 익혔다.

## 문제 3 - 정렬로 K번째 수 찾기

### 문제
N개의 수가 주어졌을 때, i번째부터 j번째까지 자른 후 정렬했을 때 k번째 수를 출력하시오.

### 풀이 아이디어
배열에서 startIndex~endIndex 구간의 원소를 List에 담은 뒤 `Collections.sort()`로 정렬했다.  
정렬된 리스트에서 `k-1` 인덱스(1-index → 0-index)의 값을 출력했다.

### 사용한 개념
- `BufferedReader`
- `List<Integer>` / `ArrayList`
- `Collections.sort()`
- 1-index → 0-index 변환

### 시간복잡도
- O(n log n)

### 공간복잡도
- O(n)

---

### 최적화 — `Arrays.copyOfRange()` 사용

```java
// 현재 코드 — 반복문으로 구간 추출
List<Integer> sortNums = new ArrayList<>();
for (int i = 0; i < nums.length; i++) {
    if (i >= startIndex - 1 && i <= endIndex - 1) {
        sortNums.add(nums[i]);
    }
}
Collections.sort(sortNums);
System.out.println(sortNums.get(k - 1));

// 최적화 코드 — Arrays.copyOfRange()로 한 줄에 구간 추출
int[] sliced = Arrays.copyOfRange(nums, startIndex - 1, endIndex);
Arrays.sort(sliced);
System.out.println(sliced[k - 1]);
```

`Arrays.copyOfRange(arr, from, to)` — from 포함, to 미포함

---

### 동작 예시

```
배열: [1, 5, 2, 6, 3]  (1-index)
i=2, j=5, k=3

구간 추출: index 1~4 → [5, 2, 6, 3]
정렬:      [2, 3, 5, 6]
k=3번째:   5

출력: 5 ✅
```

---

### 최종 코드
```java
int startIndex = arr[0];
int endIndex = arr[1];
int k = arr[2];

List<Integer> sortNums = new ArrayList<>();
for (int i = 0; i < nums.length; i++) {
    if (i >= startIndex - 1 && i <= endIndex - 1) {
        sortNums.add(nums[i]);
    }
}

Collections.sort(sortNums);
System.out.println(sortNums.get(k - 1));
```

---

### 면접 질문 1. 이 코드가 전체적으로 어떻게 동작하는지 설명해보세요.

#### 답변
입력받은 i, j, k를 각각 startIndex, endIndex, k로 저장했습니다.  
배열을 순회하면서 startIndex-1부터 endIndex-1까지의 원소를 List에 담았습니다.  
`Collections.sort()`로 정렬한 뒤 k-1 인덱스의 값을 출력했습니다.

---

### 면접 질문 2. 왜 `k-1` 인덱스를 사용했나요?

#### 답변
문제에서 i, j, k는 1-index를 사용하지만 Java 배열과 List는 0-index입니다.  
따라서 k번째 원소는 0-index 기준 k-1 위치에 있습니다.

---

### 회고
구간 추출 시 1-index를 0-index로 변환하는 것을 놓치지 않는 게 중요하다.  
`Arrays.copyOfRange()`를 사용하면 반복문 없이 구간을 한 줄로 추출할 수 있다.  
버블 정렬, 선택 정렬 모두 O(n²)이지만 실제 코테에서는 `Arrays.sort()` (O(n log n))를 쓰는 게 맞다.
