## 🟢 문제 1. 두 배열 합쳐서 정렬 `March25MergeArrays`

### 문제 설명
두 개의 오름차순 정렬된 배열이 주어졌을 때, 두 배열을 합쳐서 오름차순으로 출력하시오.

### 풀이 아이디어
처음에는 PriorityQueue에 두 배열을 모두 넣어 정렬했으나,
"이미 정렬된 배열임을 활용할 것"이라는 조건에 따라 투 포인터 병합으로 수정했다.
두 포인터 i, j를 앞에서부터 이동하며 더 작은 값을 먼저 StringBuilder에 담는다.
한 쪽 배열이 먼저 끝나면 남은 배열을 통째로 붙인다.

### 사용한 개념
- 투 포인터 병합 (Merge Sort의 핵심 단계)
- StringTokenizer, BufferedReader
- StringBuilder

### 시간복잡도 / 공간복잡도
- 시간복잡도: O(N+M) — 각 배열을 한 번씩만 순회
- 공간복잡도: O(N+M) — 결과 배열 크기

### 버그 기록
- PriorityQueue 사용 시 O((N+M) log(N+M))으로 정렬 정보를 버리게 됨
- `for (int arr : firstArr)` → `arr`이 배열이 아닌 원소임에도 배열명처럼 작성하는 실수

### 최종 코드
```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class March25MergeArrays {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int firstSize = Integer.parseInt(br.readLine());
        StringTokenizer firstTokenizer = new StringTokenizer(br.readLine());
        int[] firstArr = new int[firstSize];
        for (int i = 0; i < firstSize; i++) {
            firstArr[i] = Integer.parseInt(firstTokenizer.nextToken());
        }

        int secondSize = Integer.parseInt(br.readLine());
        StringTokenizer secondTokenizer = new StringTokenizer(br.readLine());
        int[] secondArr = new int[secondSize];
        for (int i = 0; i < secondSize; i++) {
            secondArr[i] = Integer.parseInt(secondTokenizer.nextToken());
        }

        StringBuilder sb = new StringBuilder();
        int i = 0, j = 0;

        while (i < firstSize && j < secondSize) {
            if (firstArr[i] <= secondArr[j]) {
                sb.append(firstArr[i++]);
            } else {
                sb.append(secondArr[j++]);
            }
            sb.append(" ");
        }

        while (i < firstSize) {
            sb.append(firstArr[i++]).append(" ");
        }

        while (j < secondSize) {
            sb.append(secondArr[j++]).append(" ");
        }

        System.out.println(sb.toString().trim());
    }
}
```

### 면접 Q&A
**Q. 이미 정렬된 두 배열을 합칠 때 가장 효율적인 방법은?**
A. 투 포인터 병합을 사용하면 O(N+M)으로 처리할 수 있습니다.
두 포인터를 각 배열의 앞에서부터 시작해 작은 값을 먼저 담고,
한 쪽이 끝나면 나머지를 그대로 붙입니다.
이는 병합 정렬(Merge Sort)의 핵심 단계와 동일한 패턴입니다.

**Q. `i++`와 `++i`의 차이는?**
A. `i++`는 현재 값을 먼저 사용한 뒤 증가(후위), `++i`는 먼저 증가한 뒤 사용(전위)합니다.
`sb.append(arr[i++])`는 `arr[i]`를 append한 후 i를 증가시키므로 올바르게 동작합니다.

### 회고
PriorityQueue도 정렬은 되지만 "이미 정렬된 배열"이라는 힌트를 활용하지 못했다.
조건을 꼼꼼히 읽고 O(N+M)으로 줄일 수 있는지 먼저 판단하는 습관을 들여야 한다.

## 🟡 문제 2. 수 정렬하기 (내림차순 + 중복 제거) `March25SortDescUnique`

### 문제 설명
N개의 수가 주어졌을 때, 중복을 제거하고 내림차순으로 정렬해서 출력하시오.

### 풀이 아이디어
HashSet으로 중복을 제거하면서 입력을 받고,
ArrayList로 변환 후 Collections.reverseOrder()로 내림차순 정렬한다.
N이 최대 1,000,000이므로 출력은 StringBuilder를 사용한다.

### 사용한 개념
- HashSet (중복 제거)
- Collections.sort + reverseOrder (내림차순)
- StringBuilder (대량 출력)

### 시간복잡도 / 공간복잡도
- 시간복잡도: O(N log N) — HashSet 삽입 O(N) + 정렬 O(K log K)
- 공간복잡도: O(K) — K는 중복 제거 후 원소 수

### 버그 기록
- `System.out.println(numList)` → `[3, 2, 1]` 형태로 출력되는 버그 발생
- StringBuilder로 한 줄씩 출력하도록 수정

### 최종 코드
```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.*;

public class March25SortDescUnique {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine());
        StringTokenizer st = new StringTokenizer(br.readLine());

        Set<Integer> numSet = new HashSet<>();

        for (int i = 0; i < n; i++) {
            numSet.add(Integer.parseInt(st.nextToken()));
        }
        List<Integer> numList = new ArrayList<>(numSet);
        Collections.sort(numList, Collections.reverseOrder());

        StringBuilder sb = new StringBuilder();
        for (int num : numList) {
            sb.append(num).append("\n");
        }
        System.out.print(sb);
    }
}
```

### 면접 Q&A
**Q. HashSet과 ArrayList를 조합해서 중복 제거 + 정렬을 하는 이유는?**
A. HashSet은 중복을 자동으로 제거하지만 정렬 기능이 없고,
ArrayList는 정렬은 가능하지만 중복 제거 기능이 없습니다.
두 자료구조를 조합하면 각각의 장점을 활용할 수 있습니다.

**Q. N이 1,000,000일 때 println을 루프에서 쓰면 안 되는 이유는?**
A. `System.out.println`은 매번 flush를 발생시켜 I/O 비용이 큽니다.
StringBuilder에 모아서 한 번에 출력하면 시간 초과를 방지할 수 있습니다.

### 회고
처음에 `System.out.println(numList)`로 출력 형식을 틀렸다.
List를 그대로 출력하면 대괄호와 쉼표가 포함된다는 것을 기억해야 한다.
배열을 거치지 않고 바로 Set에 넣는 방식으로 불필요한 변수를 제거했다.

## 🟡 문제 3. 가장 큰 수 만들기 `March25LargestNumber`

### 문제 설명
정수 배열이 주어졌을 때, 배열의 원소를 이어 붙여 만들 수 있는 가장 큰 수를 출력하시오.

### 풀이 아이디어
처음에 스택으로 접근했으나 잘못된 방향이었다.
핵심은 두 수를 어떤 순서로 이어 붙였을 때 더 큰 수가 되는지를 기준으로 정렬하는 것이다.
`a + b`와 `b + a`를 문자열로 비교해서 `b + a`가 더 크면 b를 앞에 놓는다.
int[]를 쓰지 않고 바로 String[]에 담아 불필요한 변환을 줄였다.

### 사용한 개념
- 커스텀 Comparator `(a, b) -> (b + a).compareTo(a + b)`
- Arrays.sort
- StringBuilder

### 시간복잡도 / 공간복잡도
- 시간복잡도: O(N log N)
- 공간복잡도: O(N)

### 버그 기록
- 초기 접근: 스택(Deque) 사용 → 완전히 잘못된 방향
- 불필요한 import `ArrayDeque`, `Deque` 방치 → 제거 필요

### 최종 코드
```java
package week3;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class March25LargestNumber {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        String[] strNums = new String[st.countTokens()];

        for (int i = 0; i < strNums.length; i++) {
            strNums[i] = st.nextToken();
        }

        Arrays.sort(strNums, (a, b) -> (b + a).compareTo(a + b));

        StringBuilder sb = new StringBuilder();
        for (String strNum : strNums) {
            sb.append(strNum);
        }

        System.out.println(sb);
    }
}
```

### 면접 Q&A
**Q. 가장 큰 수를 만들기 위한 정렬 기준은 어떻게 설계하나요?**
A. 두 수 a, b를 문자열로 이어 붙인 결과를 비교합니다.
`b + a`가 `a + b`보다 크면 b를 앞에 놓는 방식으로 Comparator를 정의합니다.
예를 들어 "3"과 "30"을 비교하면 "330" > "303"이므로 "3"이 앞에 옵니다.

**Q. 숫자 크기로 정렬하면 안 되는 이유는?**
A. 숫자 크기로 정렬하면 "30"이 "3"보다 크지만,
이어 붙이면 "303" < "330"으로 "3"이 앞에 와야 합니다.
단순 숫자 비교로는 이 케이스를 처리할 수 없기 때문입니다.

### 회고
처음에 스택으로 접근해 방향을 완전히 잘못 잡았다.
자료구조를 먼저 꺼내는 습관이 남아있는 것 같다.
"어떤 기준으로 정렬할까"를 먼저 생각하는 습관을 들여야 한다.
커스텀 Comparator의 작동 원리(compareTo 반환값과 정렬 순서의 관계)를 잘 이해했다.

## 오늘의 핵심 개념 정리

| 개념 | 내용 |
|---|---|
| 투 포인터 병합 | 정렬된 두 배열 → O(N+M)으로 합치기. Merge Sort의 핵심 단계 |
| 후위 증가 `i++` | 현재 값 사용 후 증가. `arr[i++]`은 arr[i] 사용 뒤 i 증가 |
| 커스텀 Comparator | `(a, b) -> (b+a).compareTo(a+b)` — 이어 붙인 결과 기준 정렬 |
| compareTo 반환값 | 양수 → b 앞에, 음수 → a 앞에, 0 → 유지 |
