## 🟡 문제 1. 가장 많이 등장한 문자

### 문제 설명

문자열이 주어졌을 때, 각 문자가 몇 번 등장하는지 세어서 가장 많이 등장한 문자를 반환하라.
등장 횟수가 같은 문자가 여러 개라면 알파벳 순서가 빠른 것을 반환한다.

- 입력: `"banana"`
- 출력: `a`
- 제약 조건: 1 ≤ N ≤ 100,000 / 소문자 알파벳만 포함

---

### 문제 요약

- 한 줄 요약: 문자열에서 가장 많이 등장한 문자 출력, 동점이면 알파벳 순 앞선 것
- 입력 / 출력 구조: 문자열 1개 → 문자 1개
- 헷갈리는 조건: 동점 처리 (알파벳 순 우선)

---

### 풀이 전 제출 

```
1. 입력 크기 (N): 100,000
2. 목표 시간복잡도: O(n)
3. 사용할 알고리즘 후보: 알파벳 26개짜리 int 배열
4. 선택한 이유: 알파벳은 고정 범위 → c - 'a' 로 O(1) 인덱스 접근 가능
```

---

### 풀이 아이디어

1. `int[] alphabetCount = new int[26]` 선언
2. 문자열 순회하며 `alphabetCount[c - 'a']++`
3. 배열 순회하며 최댓값 인덱스 추적 (`alphabetCount[maxIndex]`와 직접 비교)
4. `(char)(maxIndex + 'a')` 로 결과 문자 변환 후 출력

알파벳 순 우선 처리는 배열을 0(a)부터 순회하고 **초과(>)** 조건만 사용하면 자동 보장됨

---

### 테스트 케이스 설계

| 케이스    | 입력         | 예상 출력 | 결과 |
|--------|------------|-------|----|
| 최솟값    | `"a"`      | `a`   | ✅  |
| 동점 케이스 | `"abcd"`   | `a`   | ✅  |
| 기본 케이스 | `"banana"` | `a`   | ✅  |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n) — 문자열 순회 1회 + 알파벳 배열 순회 26회(상수)
- 공간복잡도: O(1) — 크기 고정 배열 26개

---

### 버그 기록

- `maxValue` 갱신 누락: `maxIndex`만 갱신하고 `maxValue`를 업데이트하지 않아 최댓값 추적 불가
- 수정: `maxValue = alphabetCount[i]` 추가 → 이후 `maxValue` 변수
  제거하고 `alphabetCount[maxIndex]`로 직접 비교하도록 리팩토링

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Apr16Problem1 {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String input = br.readLine();
        int[] alphabetCount = new int[26];

        for (int i = 0; i < input.length(); i++) {
            char c = input.charAt(i);
            int index = c - 'a';
            alphabetCount[index]++;
        }

        int maxIndex = 0;
        for (int i = 1; i < alphabetCount.length; i++) {
            if (alphabetCount[i] > alphabetCount[maxIndex]) {
                maxIndex = i;
            }
        }

        char result = (char) (maxIndex + 'a');
        System.out.println(result);
    }
}
```

### 💡 int[] + 문자 인덱싱 패턴

**장점:**
알파벳 26개처럼 범위가 고정된 경우 `c - 'a'` 한 줄로 O(1) 인덱스 접근 가능. HashMap보다 빠르고 메모리도 적게 씀.

**단점:**
범위가 고정된 경우에만 사용 가능. 유니코드나 범위가 넓으면 배열 크기가 비현실적으로 커짐.

**실무 사용처:**
문자 빈도 분석, 언어 감지, 암호화 알고리즘 등 고정 범위 문자 처리.

---

### 패턴 노트

```
## Apr16Problem1 - 가장 많이 등장한 문자

- 문제 유형: 빈도 분석
- 핵심 패턴: c - 'a' 로 알파벳을 인덱스로 변환 → int[] 배열에 빈도 저장
- 내가 놓친 포인트: maxValue 갱신 누락 / maxIndex만 갱신하면 비교 기준이 고정됨
- 다음에 나오면 접근 방식: 고정 범위 문자 → int[] 배열, 가변 범위 → HashMap
```

## 🟡 문제 2 (시뮬레이션). 번호표 교환

### 문제 설명

N명의 학생이 번호표(1번~N번)를 들고 있다.
홀수 번호 학생은 자신의 번호에 1을 더한 번호 학생과 교환한다.
N이 홀수이면 마지막 학생은 교환하지 않는다.
교환을 M번 반복한 후 최종 번호 배열을 출력하라.

- 입력: `5 3`
- 출력: `2 1 4 3 5`
- 제약 조건: 1 ≤ N ≤ 100 / 1 ≤ M ≤ 1,000

---

### 문제 요약

- 한 줄 요약: 홀수 인덱스 학생과 다음 학생의 번호를 M번 교환, N 홀수면 마지막 제외
- 입력 / 출력 구조: N, M → 번호 배열
- 헷갈리는 조건: N이 홀수일 때 마지막 학생 처리

---

### 풀이 전 제출 

```
1. 입력 크기 (N): 100
2. 목표 시간복잡도: O(n)
3. 사용할 알고리즘 후보: int[] 배열 + 반복문
4. 선택한 이유: 규칙을 그대로 순서대로 구현
```

---

### 풀이 아이디어

1. `int[] nums = new int[n]` 선언, `nums[i] = i + 1` 초기화
2. M번 반복 → 홀수 번호 위치에서 다음 위치와 swap
3. **최적화 포인트:** 1회전과 2회전이 서로 역연산 → M이 짝수면 초기 상태와 동일
    - `boolean isChange = m % 2 == 1` 으로 최대 1번만 루프

---

### 테스트 케이스 설계

| 케이스         | 입력         | 예상 출력       | 결과 |
|-------------|------------|-------------|----|
| M=0 (변화 없음) | `5 0`      | `1 2 3 4 5` | ✅  |
| N 홀수        | `5 1`      | `2 1 4 3 5` | ✅  |
| N 짝수        | `4 1`      | `2 1 4 3`   | ✅  |
| M 최댓값 짝수    | `100 1000` | 초기 상태 그대로   | ✅  |

---

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n) — M % 2 최적화로 루프 최대 1회
- 공간복잡도: O(n) — 번호 배열

---

### 버그 기록

- 인덱스 범위 오류: `new int[n+1]`로 선언 후 0번 인덱스 미사용 → `new int[n]`으로 수정, 인덱스 0부터 시작
- `for (int i = 0; i <= n; i++)` 출력 시 범위 초과 → `i < n` 으로 수정

---

### 최종 코드

```java
package week6;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Apr16Simulation1 {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] input = br.readLine().split(" ");
        int n = Integer.parseInt(input[0]);
        int m = Integer.parseInt(input[1]);
        int[] nums = new int[n];

        for (int i = 0; i < n; i++) {
            nums[i] = i + 1;
        }

        boolean isChange = m % 2 == 1;

        if (isChange) {
            for (int i = 0; i < n; i++) {
                int current = i + 1;
                if (current == n && current % 2 == 1) {
                    break;
                }
                if (current % 2 == 1) {
                    int temp = nums[i];
                    nums[i] = nums[i + 1];
                    nums[i + 1] = temp;
                }
            }
        }

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) {
                sb.append(" ");
            }
            sb.append(nums[i]);
        }

        System.out.println(sb);
    }
}
```

**장점:**

**단점:**

**실무 사용처:**
게임 서버의 턴 처리, 배치 작업 스케줄링, 이벤트 큐 처리.
