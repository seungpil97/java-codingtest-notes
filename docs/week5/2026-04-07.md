## 문제 1. 애너그램 그룹 🟡

### 문제 설명

문자열 배열이 주어진다. 서로 애너그램인 문자열끼리 묶어서 출력하라.
각 그룹은 원래 입력 순서 기준으로 처음 등장한 순서대로 출력하고, 그룹 내 문자열도 입력 순서를 유지한다.

### 문제 요약 (내 언어로 재정의)

- 한 줄 요약: 주어진 문자열 배열 중 서로 애너그램인 문자열끼리 묶어서 순서대로 출력
- 입력 / 출력 구조: 입력 N개 문자열 / 출력 그룹별 문자열
- 애너그램: 같은 문자들로 구성된 문자열 (철자를 재배열하면 서로 같아지는 문자열)

### 풀이 전 제출

1. 입력 크기 (N): 100,000
2. 목표 시간복잡도: O(N) (문자열 길이 L ≤ 20 상수이므로 O(N * L log L) = O(N))
3. 사용할 알고리즘 후보: HashMap
4. 선택한 이유: 정렬된 문자열을 key로 사용해서 같은 그룹끼리 묶기 위해

### 풀이 아이디어

- 각 문자열을 `toCharArray()` → `Arrays.sort()` → `new String()` 으로 key 생성
- `LinkedHashMap<String, List<String>>`에 key 기준으로 그룹화
- `map.values()` 순회하며 `String.join(" ", group)` 출력

### 사용한 개념

- `toCharArray()` + `Arrays.sort()` + `new String()` 조합
- `LinkedHashMap` — 삽입 순서 보장
- `getOrDefault()` 패턴
- `String.join()`

### 시간복잡도 / 공간복잡도

- 시간: O(N * L log L) = O(N) (L ≤ 20 상수)
- 공간: O(N)

### 최종 코드

```java
public class April07AnagramGroup {

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    br.readLine(); // N 무시
    String[] input = br.readLine().split(" ");

    Map<String, List<String>> map = new LinkedHashMap<>();

    for (String s : input) {
      char[] chars = s.toCharArray();
      Arrays.sort(chars);
      String key = new String(chars);

      List<String> group = map.getOrDefault(key, new ArrayList<>());
      group.add(s);
      map.put(key, group);
    }

    for (List<String> group : map.values()) {
      System.out.println(String.join(" ", group));
    }
  }
}
```

### 💡 LinkedHashMap

**장점:**
`HashMap`의 O(1) 탐색 속도를 유지하면서 삽입 순서를 보장한다. 내부적으로 이중 연결 리스트를 추가로 관리하기 때문에 순서가 필요한
상황에서 별도 정렬 없이 처리 가능하다.

**단점:**
이중 연결 리스트를 추가로 유지하므로 `HashMap`보다 메모리를 더 사용한다. 순서가 필요 없다면 `HashMap`이 더 효율적이다.

**실무 사용처:**

- Spring MVC의 HTTP 요청 파라미터 순서 유지
- LRU 캐시 구현 시 `removeEldestEntry()` 오버라이드 활용
- Jackson 등 JSON 직렬화 시 필드 순서 보장

### 회고

- `HashMap` → `LinkedHashMap` 차이를 직접 겪으며 학습
- 리스트형 value는 꺼내서 add 후 다시 put해야 한다는 것 확인

## 문제 2. 길이 K인 윈도우의 최댓값 🟡

### 문제 설명

정수 배열과 윈도우 크기 K가 주어진다. 크기 K인 윈도우를 왼쪽에서 오른쪽으로 한 칸씩 이동할 때, 각 윈도우에서의 최댓값을 순서대로
출력하라.

### 문제 요약 (내 언어로 재정의)

- 한 줄 요약: 주어진 정수 배열에서 오른쪽으로 한 칸 이동할 때마다 최댓값을 순서대로 출력
- 입력 / 출력 구조: 입력 N과 K, 배열 / 출력 N-K+1개 최댓값

### 풀이 전 제출

1. 입력 크기 (N): 100,000
2. 목표 시간복잡도: O(N log N)
3. 사용할 알고리즘 후보: PriorityQueue
4. 선택한 이유: 항상 최댓값을 O(log N)에 꺼낼 수 있고, (값, 인덱스) 저장으로 범위 밖 원소 제거 가능


### 풀이 아이디어

- `PriorityQueue<int[]>`에 `(값, 인덱스)` 저장, 값 기준 내림차순 정렬
- 윈도우 완성 시점(i >= k-1)마다 `peek()[1]`로 인덱스 확인
- 범위 밖(`peek()[1] < i - k + 1`)이면 `poll()`로 제거
- 남은 최댓값 출력

### 사용한 개념

- `PriorityQueue<int[]>` — 값과 인덱스 묶어서 저장
- `(a, b) -> b[0] - a[0]` — 내림차순 Comparator
- 슬라이딩 윈도우 패턴
- 범위 밖 원소 지연 제거 패턴

### 시간복잡도 / 공간복잡도

- 시간: O(N log N) — 각 원소 삽입/삭제 O(log N)
- 공간: O(N) — pq에 최대 N개 저장

### 최종 코드

```java
public class April07WindowMax {

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String[] input = br.readLine().split(" ");
    int n = Integer.parseInt(input[0]);
    int k = Integer.parseInt(input[1]);

    StringTokenizer st = new StringTokenizer(br.readLine());
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> b[0] - a[0]);

    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < n; i++) {
      pq.offer(new int[]{Integer.parseInt(st.nextToken()), i});

      if (i >= k - 1) {
        if (i > k - 1) sb.append(" ");
        while (pq.peek()[1] < i - k + 1) {
          pq.poll();
        }
        sb.append(pq.peek()[0]);
      }
    }

    System.out.println(sb);
  }
}
```

### 💡 PriorityQueue (우선순위 큐)

**장점:**
최댓값 또는 최솟값을 항상 O(log N)에 꺼낼 수 있다. 내부가 힙(Heap) 구조라 삽입/삭제 모두 O(log N)으로 유지된다.

**단점:**
특정 위치의 원소를 직접 삭제하는 것이 O(N)이다. 이 문제처럼 범위 밖 원소를 제거할 때 인덱스를 함께 저장하고 peek 시점에 걸러내는
방식으로 우회해야 한다.

**실무 사용처:**

- 카카오 알림 시스템 — 우선순위 높은 알림 먼저 처리
- Java `ThreadPoolExecutor` — 작업 우선순위 기반 스케줄링
- 다익스트라 알고리즘 — 최단 거리 노드 먼저 꺼내기

### 회고

- `PriorityQueue<int[]>` 에 (값, 인덱스) 묶어서 저장하는 패턴 학습
- 윈도우 시작점 공식: `i - k + 1` — k가 달라져도 항상 동작하는 일반식

## 문제 3. 조건을 만족하는 가장 짧은 부분 배열 🔴

### 문제 설명

양의 정수 배열과 목표값 target이 주어진다. 합이 target 이상인 부분 배열 중 길이가 가장 짧은 것의 길이를 출력하라. 조건을
만족하는 부분 배열이 없으면 0을 출력한다.

### 문제 요약 (내 언어로 재정의)

- 한 줄 요약: 양의 정수 배열에서 합이 target 이상인 부분 배열 중 가장 짧은 길이 출력
- 입력 / 출력 구조: 입력 target, N, 배열 / 출력 정수 1개

### 풀이 전 제출

1. 입력 크기 (N): 100,000
2. 목표 시간복잡도: O(N)
3. 사용할 알고리즘 후보: 투 포인터
4. 선택한 이유: 양의 정수이므로 배열 전체를 한 번만 순회 가능


### 풀이 아이디어

- `right`로 범위를 늘리며 sum에 더함
- `sum >= target`이면 `minLen` 갱신 후 `left`를 줄이며 최적화
- 끝까지 순회 후 `minLen`이 `MAX_VALUE`면 0 출력

### 사용한 개념

- 투 포인터 패턴 (left, right)
- `Integer.MAX_VALUE` 초기값 설정
- `Math.min()` 으로 최솟값 갱신

### 시간복잡도 / 공간복잡도

- 시간: O(N) — left, right 각각 최대 N번 이동
- 공간: O(1)

### 버그 기록

- `minLen = 0` 초기화 → `Integer.MAX_VALUE` 로 수정
- `if (right >= target)` → `if (sum >= target)` 으로 수정 (right는 인덱스)
- while 안에 불필요한 `if` 중첩 제거

### 최종 코드

```java
public class April07MinSubarray {

  static int[] arr;

  public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String[] input = br.readLine().split(" ");
    int target = Integer.parseInt(input[0]);
    int n = Integer.parseInt(input[1]);

    StringTokenizer st = new StringTokenizer(br.readLine());
    arr = new int[n];
    for (int i = 0; i < n; i++) {
      arr[i] = Integer.parseInt(st.nextToken());
    }

    int left = 0;
    int right = 0;
    int sum = 0;
    int minLen = Integer.MAX_VALUE;

    for (int i = 0; i < n; i++) {
      sum += arr[right];
      right++;

      while (sum >= target) {
        minLen = Math.min(minLen, right - left);
        sum -= arr[left];
        left++;
      }
    }

    System.out.println(minLen == Integer.MAX_VALUE ? 0 : minLen);
  }
}
```

### 💡 투 포인터

**장점:**
left, right 두 포인터가 각각 한 번씩만 이동하므로 전체 순회 횟수가 최대 2N번, O(N)이 보장된다. 브루트포스 O(N²) 대비
N = 100,000에서 10만 번 vs 100억 번 차이다.

**단점:**
배열 원소가 양의 정수일 때만 동작한다. 음수가 포함되면 left를 줄여도 sum이 감소하지 않아 while 종료 조건이 보장되지 않는다.

**실무 사용처:**

- 네트워크 패킷 버퍼에서 특정 크기 이상의 연속 데이터 탐색
- 데이터베이스 range scan에서 조건 만족 구간 탐색

### 회고

- 투 포인터가 양의 정수 조건에서만 동작한다는 제약 조건 확인
- `Integer.MAX_VALUE` 초기값 패턴 — "아직 못 찾았다"는 의미로 사용
- `right - left` 가 현재 윈도우 길이인 이유: right는 이미 +1 된 상태이므로 별도 +1 불필요
